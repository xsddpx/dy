# 辅助流程：评论自动回复

## 文档边界

- 本模块独立处理抖音和快手的评论查看、筛选、回复和记录，仅在用户明确要求时执行。
- 评论任务不属于默认 `auto/fast` 链路，不修改生成、衣柜、prompt 或发布规则。

## 入口

- 项目固定 CDP Chrome：`http://127.0.0.1:9222`。
- 抖音评论管理页：`https://creator.douyin.com/creator-micro/interactive/comment`。
- 快手评论管理页：`https://cp.kuaishou.com/article/comment`。如页面结构变化，优先从 `https://cp.kuaishou.com/` 创作者中心内的互动管理 -> 评论管理入口进入；未确认页面结构前不得硬猜提交接口。
- 进入页面前先读取 `AGENTS.md`、`DOCS/PROJECT.md` 和本文。
- 如 CDP、Playwright、登录态、权限、验证码、账号安全或网络代理阻断，停止评论回复任务并按 `AUX_ENV_REPAIR.md` 处理环境问题。

## 处理范围

- 只处理能明确判断为尚未回复的新评论；回复状态无法确认时跳过并记录原因。
- 默认目标是最大化安全回复覆盖率：除硬跳过项和状态不明项外，每条明确未回复评论都应优先生成并提交一条短回复，不因内容过短、只有表情、贴纸或语义轻微含糊而轻易跳过。
- 默认从各平台评论管理页当前可见作品开始；如果默认作品显示 `暂无更多评论` 或同类空状态，不得直接判定全账号无评论，应检查作品列表中评论数大于 0 的作品。
- 回复前读取该评论当前可见的完整文字、父评论和已有回复；用户已确认本项目评论者均为认识的熟人，可采用自然熟人语气，但不从用户名、头像或主页推断具体关系、共同经历和个人偏好。
- 不处理私信、粉丝群、作品删除、作品公开状态变更、置顶、拉黑或举报操作。

## 硬跳过

以下评论不回复，直接跳过并记录原因；未落入这些边界的评论默认优先回复：

- 辱骂、人身攻击、恶意挑衅。
- 广告、引流、联系方式、私下交易。
- 擦边诱导、低俗要求、性暗示升级。
- 敏感政治、违法违规、危险行为。
- 未成年人相关内容。
- 上下文严重不足，且连中性接话、卖关子或安全轻追问都可能造成误解或风险。

## 回复口吻

- 回复符合 Anna 的成年轻熟穿搭人设，语气自然、俏皮、可爱且克制。
- 用户已确认评论者均为认识的熟人，回复默认比普通公开互动更熟络、轻暧昧，可用害羞、卖关子、俏皮反问和轻微撩拨接话；不升级为露骨性暗示、具体关系承诺或线下邀约。
- 使用日常白话、口语碎片和适度网络用语，根据评论自然表达开心、害羞、惊讶或调侃，避免客服腔、模板句和长篇解释。
- 回复长度和情绪强度跟随对方：单个表情或极短评论用短接话，普通夸赞通常用一个短句，具体问题可用一至两句说明；不把轻描淡写的评论强行回复得过度热情。
- 优先接住评论里最明确的一个词或意思，再从感谢、共情、调侃、回答、轻追问和自然收住中选择最合适的一种；不在一句话里堆叠多个互动目的。
- 语气词、标点和表情按语义自然使用，默认最多一个 emoji 或颜文字，不连续堆叠感叹号、波浪号和表情。
- 只有追问确实有助于理解或延续当前话题时才使用问句，不把每条回复都写成拉互动的问题。
- 安全但含义不清时可简短追问，如 `是说这套吗`；缺少上下文或语义风险高时跳过。
- 纯表情、贴纸、无文字或句意不完整但没有明确风险时，优先用不承诺、不解释内容的熟人式短接话，例如 `暗号收到啦`、`你又调皮了呀`、`嘿嘿，我懂一点点`；只有无法形成安全中性回复时才跳过。
- 轻微暧昧或普通亲昵表情不等于硬跳过，可用害羞、打趣或轻轻收住的方式回应；出现明确低俗要求、性暗示升级或诱导时仍必须跳过。
- 回复不暴露自动化或 AI 身份，不承诺私下联系、不引导站外沟通，不扩大低俗暗示。

## 回复生成顺序

每条评论按以下顺序独立处理，最终只提交回复正文，不在页面输出分类或分析过程：

1. 判断主要意图：夸赞、提问、玩笑、建议、情绪表达、纯表情或无法判断。
2. 找出一个可接的语义点；没有明确语义点时使用简短情绪回应，不强行补充新话题。
3. 选择一个回复目标：接住情绪、回答问题、轻微调侃、必要追问或自然收住。
4. 写出与原评论长度和情绪相称的口语短句。
5. 对照事实边界、安全边界和近期回复做最后检查；存在风险时改写或跳过。

## 事实与承诺边界

- 只陈述页面当前可见内容或项目中能够确认的事实；不得编造品牌、价格、购买链接、真实地点、个人拥有或使用经历、拍摄经历和私下生活细节。
- 评论回复不得明确确认或否认作品、画面或人物是否由 AI 生成，也不得解释生成方式。遇到“是真人吗”“是不是 AI”“怎么生成”等评论时，可用 `你猜呀`、`嘿嘿，先保密`、`保留一点神秘感` 等简短模糊话术轻暧昧接住，但不得出现能够明确推出 AI 生成事实的表述。本规则只约束评论互动，不改变发布流程中必须设置的 `内容由AI生成` 平台声明。
- 信息不确定时自然说明不确定，例如 `这个我还真不确定`、`链接我这里没有准的`；无需为了显得热情而猜测。
- 不使用 `下次一定拍`、`马上安排`、`回头发你链接` 等无法保证兑现的承诺。安全的建议可以用 `我先记下这个方向` 等非承诺表达自然接住。
- 可基于用户已确认的熟人关系使用自然熟人语气，但不得虚构具体共同经历、关系标签、私下生活或未在当前上下文出现的历史互动。

## 批次与历史去重

- 生成新回复前查看本批次已经提交的回复，避免连续使用相同开头、相同句式或相同结尾；同一批次不提交完全相同的回复。
- 不通过机械替换近义词强行制造差异。若最自然的回应与近期内容相近，优先换接话角度或缩短句子；跨天偶尔重复常用短句是允许的。
- 最近多条回复已经集中使用 `哈哈`、`收到`、`懂你意思`、`太会夸了` 等口头禅时，后续优先直接回应评论语义，避免形成固定签名句。
- 跨次历史保存在 `MATERIAL/comment-reply-history.jsonl`。文件不存在时由首次任务创建；每行记录 `created_at`、`platform`、`work_key`、`comment_key`、`intent`、`comment_summary`、`decision`、`reply` 和 `reason`。
- `work_key` 和 `comment_key` 优先使用平台页面提供的稳定标识；没有稳定标识时，使用平台、页面可见作品标识、规范化评论文字和评论时间生成本地哈希，不把用于哈希的完整评论写入历史。
- 历史只保存简短评论摘要，不保存用户名、头像、主页、联系方式或完整评论。任务开始时读取最近 `50` 条记录，其中 `replied` 用于降重，`replied` 和 `unknown` 用于防重复；不把历史用于推断评论者身份或偏好。
- `decision` 使用 `replied`、`skipped`、`unknown` 或 `failed`。只有页面已确认回复成功时记为 `replied`；历史中已存在同一 `comment_key` 的 `replied` 或 `unknown` 时不再自动提交，防止重复回复。

## 推荐回复形态

- 以下短句仅用于说明语气和结构，不能作为逐条套用的话术库。
- 夸赞类：自然感谢，例如 `谢谢喜欢`、`哈哈收到`、`有被夸到`。
- 问穿搭类：只做简短回应，例如 `这套偏修身日常款`、`主要是显腰线`。
- 问场景类：简短说明画面信息，例如 `这条是暖白影棚`。
- 普通或俏皮互动：`懂你意思`、`哈哈被你发现了`、`太会夸了吧`。
- 熟人轻暧昧互动：`你这么夸我会当真的呀`、`再夸我可要飘了`、`这都被你注意到了`。
- 真人或 AI 相关提问：只卖关子，不回答事实，例如 `你猜呀`、`嘿嘿，先保密`、`保留一点神秘感`。
- 纯表情、贴纸或短句：优先简短接住，例如 `暗号收到啦`、`你又调皮了呀`、`这一下我接住了`。
- 需要延续互动时：`是说开衫这套吗`、`下次想看哪种感觉呀`。
- 自然收住：`哈哈懂你意思`、`这个角度确实蛮有感觉`、`收到啦 记住了`。

## 执行步骤

1. 依次进入抖音和快手评论管理页，等待页面稳定。
2. 检查是否登录、是否存在验证码、账号安全或权限阻断。
3. 读取 `MATERIAL/comment-reply-history.jsonl` 最近 `50` 条记录；文件不存在时按空历史继续。
4. 查看当前作品评论区；如为空，检查作品列表并切换到评论数大于 0 的作品。
5. 对每条可见评论判断是否为尚未回复，并读取当前可见对话上下文。
6. 先尝试为每条非硬跳过评论生成安全短回复；表情、贴纸和含义轻微模糊的内容优先使用中性熟人式接话。只有命中硬跳过边界或无法形成安全回复时才记录跳过原因；其余回复完成批次去重和事实检查后提交。
7. 提交前再次确认目标评论未被回复，提交后确认页面显示已发送或评论状态已变化；无法确认则记录为未知，不重复提交。
8. 每处理一条评论后，将最小必要记录追加到历史文件；页面状态与历史记录冲突时以避免重复提交为优先。
9. 完成当前可处理范围后停止，不无限翻页、不批量扫全账号，除非用户明确要求扩大范围。

## 停止条件

- 当前可见范围没有能明确判断为尚未回复的新评论。
- 遇到登录失效、验证码、账号安全、权限或页面关键控件不可用。
- 回复状态无法确认，且继续操作可能重复回复。
- 页面出现异常提交失败或平台限制。
- 用户要求停止。
