# 项目说明

## 核心目标

本项目是 `anna auto/fast` 日更发布项目，目标是稳定生成并同步发布单人竖屏短视频到抖音和快手。

- 固定路线：`anna` 单人。
- 固定模式：`auto/fast`，其中 `auto` 就是 `fast`。
- 视频规格：9:16、720p、5-6 秒。
- 生成工具：Dreamina CLI 负责视频。
- 发布渠道：抖音创作者中心和快手创作者中心，两个平台发布前都必须设置 `自主声明/作品声明 -> 内容由AI生成`。

## 核心卖点

未经用户明确允许，不得修改本节核心卖点。

- 我们的赛道是：成年轻熟女 Anna 的成熟性感修身穿搭短视频，突出夸张身材曲线、上身辨识度和腰线，用生活化互动动作做克制擦边吸引。
- 核心赛道词：御姐/轻熟性感/纯欲反差/修身吊带/上身丰满/身材比例/腰线穿搭
- 受众定位：成年男性强性吸引偏好用户。
- 视觉卖点：夸张成熟身材、巨大胸部视觉体量、成熟曲线、修身穿搭、清晰腰线、上身辨识度。
- 风格调性：纯欲反差、擦边吸引力、轻熟性感、克制诱惑感、活泼互动感与生活化动作。

## 内容方向

- Anna 是喜欢旅行、穿搭、逛街、吃饭、购物、休息和轻运动的成年女性；默认复用当前 active 一周行程资产，按当天日期、季节、天气、城市片区、白天/晚上双时段活动和旅行状态决定主题大方向。
- 每次生成前必须在当天行程内选择一个明确生活事件，例如购物、吃饭、逛街、酒店休息、跑步/轻运动、出门、交通中转或夜间小酒馆，让画面像真人旅行生活中的一个片段。
- 执行者自主设计本次作品的画面方案、场景、动作、镜头、卖点、发布文案方向和互动方式，并先想清楚本条视频的新鲜点：场景、动作、手部任务、身体方向、视线关系或镜头距离中至少有一两项体现当次差异；再从 Anna 衣柜优先匹配最合适的款式。
- 表情、视线和脸部状态随生活动作、镜头距离和场景自然变化；看镜头、侧脸、低头看手部动作、看向环境、看向镜中手机、回到平静神态等都可作为当次创作选择。
- 行程地点不是 prompt 场景的硬锁，具体画面可延展到酒店、机场、游泳馆、网球馆、商场、咖啡馆、餐厅、街区、酒吧、夜店等与当天行程合理相关的场景。
- prompt 只写可见画面、动作和镜头语言，不写合规说明；发布文案必须保持平台可发布表达。严禁生成低俗、裸体及未成年感内容。

## 内容边界

- 只写可见画面、动作、镜头、场景、穿搭和画面质感。
- 严禁低俗、裸体、未成年感和不可发布内容。
- 只执行 `anna auto/fast` 单图发布链路。

## 固定资产

- 固定角色图：`MATERIAL/fixed-role/anna.png`，这是一张同一位成年女性的多视角、多表情角色参考图。
- 历史素材账本仅为旧记录留存；默认 `auto/fast` 的选题与生成来源固定为 active 行程、Anna 衣柜、固定角色图和执行者自主创作设计。
- Anna 衣柜固定账本：`MATERIAL/anna-wardrobe.md`；用户明确要求加入衣柜的固定款式写入该文件，后续可作为 Anna 稳定、可重复穿着的个人衣柜来源。
- 一周行程资产：`MATERIAL/anna-weekly-itinerary.md` 和 `MATERIAL/anna-weekly-itinerary.json`；当前日期在 `valid_from` 到 `valid_to` 窗口内时持续复用，只有不可用或日期超出窗口时才从运行当天起重新生成连续 7 天行程，生成后直接保存为 `status=active` 并用于当天任务，无需用户确认。生成或重建行程时，每天不得写具体钟点，`time_slot` 统一写 `白天；晚上`，并分别写清白天和晚上可拍活动、场景、穿搭方向和主题关键词。

## 本地环境

- 固定执行账户 `xsddpx` 的 CDP Chrome：`TOOLS/open_cdp_chrome.sh`，默认 `http://127.0.0.1:9222`。
- Chrome 用户数据目录：`/Users/xsddpx/Library/Application Support/Google/Chrome-Codex-CDP`。
- CDP 接入默认优先使用 Playwright `connect_over_cdp`；AppleScript、系统文件选择器等只作为人工排障或兼容兜底。
- 模块 00 是环境修复模块，默认跳过；只有遇到 CDP、Playwright、登录态、API key、积分、权限、素材、读写、网络代理等环境问题，或用户明确要求修环境时才执行。
- auto/fast 视频：`dreamina multimodal2video` 只上传 `MATERIAL/fixed-role/anna.png`，`--model_version seedance2.0_vip --video_resolution 720p --duration 5|6`。

## 目录边界

- `TEMP/`：过程文件，可清理，不作为默认续跑状态。
- `OUTPUT/`：正式成片，扁平化保存为 `OUTPUT/RUN_ID.mp4`。
- `DOCS/`：流程和规则。
- `TOOLS/`：自动化脚本。
- `MATERIAL/`：固定角色素材、Anna 衣柜固定账本、当前 active 一周行程资产和历史记录留存文件。
- `DOCS/MODULES/MODULE_00_ENV_REPAIR.md`：环境修复与最佳实践沉淀；默认日更跳过，遇到环境问题时才读取。

## 默认 auto/fast 流程

1. 建档：创建 `TEMP/RUN_ID/` 并初始化运行记录；默认不执行模块 00，只有遇到环境问题或用户明确要求修环境时才读取 `MODULE_00_ENV_REPAIR.md`。
2. 行程窗口：读取当前 active 的 `MATERIAL/anna-weekly-itinerary.json` 并按当天日期匹配行程；资产缺失、无效、当天日期早于 `valid_from` 或已超过 `valid_to` 时，才按今天起连续 7 天重新规划国内真实城市行程，直接保存到 `MATERIAL/` 后继续。
3. 自主选题与创作设计：按当天行程大方向先确定本次生活事件，再由执行者自行设计作品的画面方案、场景、动作、镜头、卖点、发布文案方向和互动方式；设计必须优先服务当前赛道的修身穿搭、腰线、上身辨识度、成熟曲线和生活化互动，并明确本条视频相对常规穿搭展示的新鲜点。
4. 衣柜优先选款与 grid-prompt 写作：根据本次创作设计，从 `MATERIAL/anna-wardrobe.md` 优先选择最匹配的衣柜款式；只提取 `款式提示词：` 内容，不把季节标签、环境标签、元数据或使用建议写入最终 prompt。衣柜没有合适款式时，执行者自行设计可发布穿搭。写入十段式 `grid-prompt.txt`，它是 Dreamina v1 最终 vid prompt 的派生来源。
5. 视频提示词：用 `TOOLS/prompt_lint.py derive --mode fast` 从 `grid-prompt.txt` 生成 `vid-prompt-v1.txt` 后提交 Dreamina；prompt 使用 `@图1` 指代 `anna.png`，不得含第二张图片引用、文件名、流程说明、合规说明、平台解释或“吸收/根据某文件”的表达。
6. 视频生成：只上传 `MATERIAL/fixed-role/anna.png` 作为 `@图1` 后提交 Dreamina 视频。生成成功并下载正式 MP4 到 `OUTPUT/RUN_ID.mp4` 后，执行基础质检；默认质检通过即继续发布。只有用户明确要求“发布前确认”“只生成不发布”“本次不用发布”等暂停发布时，才展示视频、首中尾帧、vid prompt、TNS 记录和是否建议发布并等待授权。
7. 发布：通过模块 03 同步上传抖音和快手，两个平台都设置 `内容由AI生成` 声明；两个平台都返回 `published` 才判定整体发布成功，不要求发布后核对创作者中心内容列表。
8. 记录收尾：成功生成正式视频后记录产物路径和关键执行结果；发布后在运行记录中补充抖音、快手和整体发布状态并刷新记录；不把本次运行状态作为下次默认续跑依据。

## 硬阻断

- 进入自主选题、提示词、生成和发布前，必须具备有效的一周行程资产；资产缺失、无效、当天日期早于 `valid_from` 或已超过 `valid_to` 时，先重新生成今天起连续 7 天行程并保存为 active。
- 进入视频生成前，必须具备 `MATERIAL/fixed-role/anna.png`、十段式 `grid-prompt.txt` 和 `vid-prompt-v1.txt`。
- 视频生成输入合同为 `@图1` 单图；vid prompt 和 Dreamina 命令必须保持同一个图片输入。
- 视频生成因 TNS/安全拦截到 `v5` 仍未产出时停止，不发布，并报告 `v1-v5` 失败摘要。
- TNS 收敛只适用于生成工具明确返回安全拦截且没有产物；网络、登录、积分、参数、上传或超时等失败直接停止报告。
- CDP、Playwright、登录态、API key、积分、权限、素材、读写或网络代理等环境类失败时，先按模块 00 修复并记录可复用最佳实践。
- 默认自主分支生成正式 MP4 并通过基础质检后直接进入模块 03；用户要求发布前确认时，收到授权后进入模块 03。
- 任一平台发布前必须完成 `内容由AI生成` 声明。
- 单个平台出现登录失效、验证码、账号安全、平台风控、上传失败、发布按钮禁用等阻断时，记录该平台失败并继续尝试另一个平台；两个平台执行完后报告整体状态。
