# 衣柜图片化入库工作流

## 职责

本工作流根据用户本次提供的一张服装参考图，或用户明确指定的历史衣柜文档中的确定款式文字，生成穿在无头、中性、约 `#737373` 的中性中灰色哑光展示人台上的标准电商商品图，同时编写与图片一一对应的服装描述，并将两者作为衣柜条目入库。

本工作流不生成人物穿搭图或视频，不触发 xdy、xdysp、Dreamina 或平台发布。本地入库成功后使用已连接的 Google Drive 应用上传正式人台商品图。最终回复按固定字段汇报入库与 Drive 结果：Drive 成功时只提供回读确认的链接，不展示、附上或链接任何图片；没有 Drive 链接时，才按 `AGENTS.md` 和用户的明确要求交付本地图片。

## 输入与生成合同

- 新建条目默认必须由用户本次提供参考图，不从历史会话、旧运行目录或现有衣柜自行选图。只有用户明确指定某份历史衣柜文档并要求批量图片化迁移时，才允许把该文档最后有效版本中的确定款式文字作为本次来源；不得合并多个历史版本或自行追加款式。用户明确要求重新生成现有条目时，当前正式商品图可作为本次唯一编辑目标，只执行用户指定变更；若要求重建质感，则以该条现有服装描述或其原始历史款式文字为唯一生成事实源，旧正式图只用于生成后核对款式一致性，不得作为图片编辑输入。
- 每条只采用一种服装事实来源：参考图模式以原始参考图为唯一来源；历史文本模式以指定文档中该条完整款式文字为唯一来源。真人身份、脸部、头发、妆容、身材、姿势、手机、界面、场景和道具均不属于服装事实。
- 目标是来源中明确的整套服装；上装、下装、裙装、领带、腰带和袜类等明确穿搭组件必须完整保留。
- 参考图未展示或被遮挡的背面、裤脚、裙摆、袜脚和局部结构，可结合可见轮廓、相邻结构与常见服装结构做最小必要、保守且单一的补全；允许生成结果与参考图适当不同，但不得改变可见的核心品类、主要颜色和关键辨识结构。只有可见部分本身相互矛盾或无法确认目标服装组件时才停止，请用户补图或说明。
- 固定使用 `$imagegen` 技能的内置图片生成工具；参考图模式每轮只使用原始参考图，历史文本模式每轮只使用同一条原始款式文字，失败候选不得成为下一轮输入。

固定基础 prompt：

```text

分别呈现各组件在参考图中可见的真实面料物理特征：符合真实尺度且连续的纤维、纱线、织纹或针织组织，合理的厚度、克重感、柔软或挺括程度、弹性、表面粗糙度及哑光或光泽响应，以及由人台支撑和重力产生的自然垂坠、压痕与受力褶皱；不得把整套服装处理成同一种光滑材质。呈现与品类相符的普通基础接缝、缝线、包边、腰头、门襟和下摆收边；参考图明确的结构优先准确还原，被遮挡部分只做非装饰性、中性且最小必要的保守补全，不新增辨识性设计。褶皱应从肩部、腋下、肘部、腰部、人台支撑点或布料重力方向自然产生，不以随机浮雕纹理代替真实结构。

使用专业服装摄影棚光线：镜头侧前方约 45° 的大型柔光主光照出纤维、织纹、接缝和褶皱的细微明暗，较弱的正面补光保留深色服装细节，轻柔轮廓光使深色和浅色服装都与中灰色人台清晰分离；曝光均衡、色彩准确，整套服装处于清晰景深内。保留真实衣物常见的轻微纤维起伏、自然压痕和接缝阴影，形成高分辨率相机拍摄真实服装实物的效果；不出现通体无纹理的光滑曲面、蜡质或塑料反光、液态边缘、过度磨皮、CG 建模感、夸张粗糙浮雕、伪造高频噪点或过锐化光晕。

固定使用原生 9:16 竖版正面全身构图，人台居中且完整，服装主体占据画面主要区域，四周保留均匀少量留白；使用纯白色或极浅灰色干净背景。参考图未展示或被遮挡的背面、裤脚、裙摆、袜脚和局部结构，可依据可见轮廓、相邻结构与常见服装结构做最小必要、保守且单一的补全；不要添加参考图中未显示的文字、品牌、参数、装饰、手机、界面、场景道具、鞋履或无关服装组件。
```

## 图片硬门

入库前逐项对照该条原始参考图或原始款式文字：

1. 来源中明确的目标服装组件全部存在。
2. 品类、颜色、领口或肩带、袖型、层次、开合、腰线、裙裤轮廓和长度与来源一致；参考图未展示或遮挡部分按固定 prompt 做单一、保守的补全，不反向改变可见结构；历史文本未定义的属性保持普通、中性且不新增辨识性设计。
3. 每个主要服装组件都能在大面积表面、结构边缘、受力或垂坠区域看到物理一致的材质线索，不能只靠颜色区分组件；来源明确的针织、罗纹、丹宁、棉质、西装或梭织、弹力、哑光、挺括或柔软等特征均有尺度真实且连续的视觉证据，纹理不得局部消失、变形或跨组件复制。
4. 至少三个适用结构位置清晰可读，例如领口、袖窿、袖口、门襟、腰头、侧缝、裤脚或裙摆；接缝与缝线沿结构连续，不粘连、融化或漂浮。未定义的基础工艺只能做普通补全，不能成为新设计。
5. 褶皱具有明确受力起点并顺重力延伸，厚薄、挺括度和垂坠与来源中的面料一致；服装不能像硬壳、充气体或贴在人台上的平面贴纸。
6. 侧前方主光形成细微方向性明暗与接缝阴影，同时深浅色区域都保留细节。若出现大面积无纹理平滑面、塑料或蜡质高光、CG 边缘、液态边缘、假噪点、夸张浮雕或锐化光晕，直接失败；材质增强不得改变原始款式、颜色、组件、透明度或图案，不得凭空增加蕾丝、亮片、皮革、金属光泽、破洞、水洗或装饰性车线。
7. 画面为未经放大的严格 9:16 正面全身商品图，服装完整清晰，人台没有遮挡、吞并或改变结构。
8. 只有一个无头、中性、约 `#737373` 的中性中灰色哑光人台；深色和浅色服装与人台的边缘、层次和材质都清晰可辨；没有真人特征、来源未指定的文字或品牌、手机、界面、道具或无关设计。

质检固定在模型原始生成分辨率、`100%` 显示下执行，不得以放大图、锐化图或后期添加纹理后的结果作为通过依据。参考图存在遮挡或截断且需要补全时，首次生成可在基础 prompt 后追加一句中性补全事实，明确本轮采用的唯一推测品类、结构或长度。任一项不通过时丢弃结果，并重新以该条原始来源生成；只允许在基础 prompt 后追加一句针对当前唯一错误的中性事实：`仅修正本轮唯一错误：<组件或具体部位>应按原始来源呈现“<已有服装事实>”，在<检查区域>以真实尺度显示<缺失的纹理、基础缝线或自然垂坠特征>，其余服装组件、颜色、版型、人台、背景和 9:16 构图全部保持不变。`

## 原生分辨率处理

- 生成时直接要求原生 `9:16`；模型原图只有在未放大即可保留足够像素且比例仅有微小偏差时，才允许使用 `TOOLS/prepare_wardrobe_image.py` 做居中最小裁剪并转换为严格 `9:16` PNG。
- 禁止使用超分辨率、插值放大、锐化、降噪、纹理叠加或其他后处理伪造材质。任一边所需裁剪超过原边长 `2%`，或无放大裁剪后的结果低于 `720x1280` 时，丢弃候选并从原始来源重新生成。
- 正式图片只接收该工具输出或已经严格为 `9:16` 且达到最小尺寸的模型原图；工具必须以临时文件准备结果并在验证尺寸后一次性替换目标文件。
- 微裁完成后必须在结果原始分辨率下重新执行第 4、7、8 项图片硬门，确认裁剪没有截断人台、服装或必要留白。

## 服装描述

图片通过硬门后，根据该条原始参考图或原始款式文字与最终商品图共同编写一条 `款式提示词`：

- 使用单段中文，通常只用一句话，并以识别整套服装所需的最少可见事实为准：写清实际存在的全部服装组件、主要颜色和关键辨识结构；其他属性仅在有助于区分款式时写入，不逐项声明未出现的属性，不重复同一事实。
- 只记录可见服装事实，不描述人物身份、身材、姿势、人台、背景、鞋履或生成过程。
- 使用明确、单一、无条件分支的表述；通用的服装一致性与衣料自然形变约束由[主链阶段 02：选题与 Prompt](../PIPELINE/02_CONTENT_AND_PROMPT.md)的 `其他：` 段统一承担，不在款式提示词中重复。
- 图片与描述一致并符合固定格式即可入库；视频 prompt 的校验留在主链阶段 02 实际运行时执行，不在本工作流提前执行。

## 原子入库

正式目录结构固定为：

```text
MATERIAL/wardrobe-images/
└── 衣柜图-夏季-001/
    ├── 衣柜图-夏季-001.png
    ├── 服装描述.md
    └── 原始参考图.jpg         # 可选，保留用户原图扩展名
```

描述文件固定格式：

```markdown
# 衣柜图-夏季-001

- 图片：衣柜图-夏季-001.png
- 款式提示词：<简洁单段服装描述>
```

入库规则：

- 只识别 `衣柜图-季节-NNN/`；季节只能是 `春季`、`夏季`、`秋季` 或 `冬季`，`NNN` 为三位十进制季内序号。
- 各季节独立编号：该季节为空时从 `001` 开始，否则取该季节最大编号加一；不回填缺号、不复用删除编号。
- 序号只在商品图和描述全部通过后分配；失败候选不占号。
- 先在正式目录外准备并校验商品图、服装描述和可选原始参考图，再以目录重命名一次性入库；不得留下半条目。
- 新条目入库时，目标目录存在则停止并重新扫描，禁止覆盖或原地修改已入库条目。
- 用户明确要求维护现有条目时，可在编号不变的前提下原子替换商品图或 `服装描述.md`；替换前必须确认图文事实一致并通过图片硬门与条目格式校验。
- 每个条目根目录只能包含同编号 PNG、`服装描述.md` 和至多一张可选的 `原始参考图.<原扩展名>`；原始参考图必须是普通文件，不得使用目录或软链接。
- 参考图模式中，用户本次提供且实际用于生成的原始参考图按原格式保存为 `原始参考图.<原扩展名>`；历史文本模式不伪造原始参考图。候选图、旧正式图、模型原图、代理图、说明、prompt、JSON 和状态文件均不进入正式条目。
- 主链阶段 02 和 `TOOLS/select_wardrobe.py` 只验证原始参考图的命名与文件类型，不将其用于选图、prompt、哈希锁定、运行记录或最终交付。
- 任一季节的编号达到 `999` 时停止，未经用户修改合同不得改为四位编号。
- 入库后立即成为正式三图视频可选条目，不维护实验或验证状态。

## Google Drive 上传

- 只上传本次已通过硬门并完成本地原子入库的 `衣柜图-季节-NNN.png`；不上传原始参考图、服装描述、候选图或代理图。
- 使用已连接的 Google Drive 应用上传到 My Drive 根目录下固定文件夹 `衣柜图/`；文件名保持 `衣柜图-季节-NNN.png`，MIME 类型使用 `image/png`。
- 上传前列出 My Drive 根目录并按完整文件夹名精确匹配未删除的 `衣柜图`。没有同名文件夹时在根目录创建；只有一个时使用其实际文件夹 ID；存在多个同名文件夹时停止 Drive 写入并汇报重复冲突，不自行选择或再创建。
- 在已确认的 `衣柜图/` 内按完整文件名精确搜索未删除文件。新条目没有同名文件时以该文件夹 ID 作为 `parent_folder_id` 新建上传；维护现有条目且只有一个同名文件时，使用该文件 ID 原位替换内容并保持其父文件夹不变，保留 Drive 链接。
- 精确搜索返回多个同名文件时不自行选择、删除或覆盖，本次 Drive 状态记为失败并汇报重复冲突。
- 写入完成后通过文件 ID 回读元数据，核对更新前后文件 ID 相同，并核对文件名、MIME 类型、大小和实际返回的 URL；远端提供 `md5Checksum` 时一并与本地 MD5 核对，不预测或拼接链接。
- Drive 文件夹发现或创建、登录、权限、网络、配额、上传或回读失败时，保留已验证的本地入库结果，汇报原因和 `待补传`；不回滚、不触发视频或平台流程。

## 最终交付

入库成功且 Drive 上传完成后，最终回复固定包含以下信息：

```markdown
- 入库状态：已入库
- 衣柜编号：衣柜图-夏季-001
- 款式提示词：<本条目最终入库的简洁描述>
- 生成状态：成功
- Google Drive 状态：已上传
- Google Drive 链接：<回读确认的实际 URL>
```
