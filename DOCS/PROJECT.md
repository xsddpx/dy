# 项目说明

## 核心目标

本项目是 `anna auto/fast` 日更发布项目，目标是稳定生成并同步发布单人竖屏短视频到抖音和快手。

- 固定路线：`anna` 单人。
- 固定模式：`auto/fast`，其中 `auto` 就是 `fast`。
- 视频规格：9:16、720p、5-7 秒。
- 生成工具：Dreamina CLI 负责视频。
- 发布渠道：抖音创作者中心和快手创作者中心，两个平台发布前都必须设置 `自主声明/作品声明 -> 内容由AI生成`。

## 核心卖点

未经用户明确允许，不得修改本节核心卖点。

- 我们的赛道是：成年轻熟女 Anna 的成熟性感修身穿搭短视频，突出夸张 S 型身材曲线、上身辨识度和腰线，用生活化互动、成熟女性妩媚感和可见才艺表现做克制擦边吸引。
- 核心赛道词：御姐/轻熟性感/纯欲反差/修身吊带/上身丰满/身材比例/腰线穿搭
- 受众定位：成年男性强性吸引偏好用户。
- 视觉卖点：夸张成熟身材、S 型身材曲线、巨大胸部视觉体量、成熟曲线、修身穿搭、清晰腰线、上身辨识度。
- 风格调性：纯欲反差、擦边吸引力、轻熟性感、克制诱惑感、成熟女性的妩媚感、活泼互动感、可见才艺表现与生活化动作。

## 内容方向

- Anna 是喜欢旅行、穿搭、逛街、吃饭、购物、休息和轻运动的成年女性；默认复用当前 active 一周行程资产，按当天日期、季节、天气、城市片区、白天/晚上双时段活动和旅行状态决定主题大方向。
- 每次生成前必须先从 `早上`、`上午`、`下午`、`晚上` 中随机确定一个拍摄时段，再选择拍摄语境、光线和画面方案；`早上`、`上午`、`下午` 归入行程资产的白天方向，`晚上` 归入晚上方向。除非用户明确指定时段，不得因夜景、玻璃反光或灯光氛围稳定而连续偏向晚上。
- 每次生成前必须在当天行程内确定一个拍摄语境或展示舞台，例如酒店窗边、商场连廊、街区灯光、海边步道、泳池外廊、餐厅门口或交通中转空间；行程用于提供真实场景、季节、光线和旅行状态，不要求每条都由购物、吃饭、房卡、端咖啡、整理行李等生活动作驱动。
- 执行者必须先以短视频创作者视角设计本次作品的卖点展示结构，再确定画面方案、场景、动作、镜头、发布文案方向和互动方式；写 prompt 前先想清楚本条视频如何更好展示 S 型身材、上身轮廓、腰线、臀胯比例、穿搭版型和成熟女性妩媚感。生活动作只作为支撑卖点的可选道具或过渡，不得盖过镜头展示主轴。
- 本条视频的新鲜点优先来自卖点呈现方式：侧身曲线、转身走近、腰线停顿、前后景深、镜头距离、拍摄关系、轻舞律动、试衣走位、肩颈展开、腰胯重心变化或视线调度；生活语境、手部展示和道具互动只有在能强化卖点时才作为新鲜点。
- 镜头设计以展示 S 型身材、穿搭轮廓、腰线和上身辨识度为主；拍摄关系可按画面需要选择自拍、固定拍摄或他人手持拍摄，不默认绑定单一视角。不同镜头关系必须符合自身特性：固定拍摄不得写成跟拍、后退或推近；他人手持拍摄可以有轻微稳定手持感，但不默认主动移动；自拍只能体现人物持机带来的轻微角度变化。除人物走路、走近、走位或试衣走位外，不需要用镜头移动制造变化。
- 每条内容可把一种可见才艺或表现力作为新鲜点之一，例如轻舞律动、穿搭搭配、镜前造型、轻运动/拉伸、试衣走位、手部造型、道具互动或镜头表现力；才艺必须通过动作、节奏、姿态或道具任务被看见，不写空泛能力说明。
- 表情、视线和脸部状态随卖点展示动作、镜头距离和场景自然变化，但默认要比静态摆拍更活泼、更有层次，并体现成熟女性的妩媚感；每条视频至少设计两次可见微表情变化，避免全程平静、全程单一微笑或机械眨眼。可使用轻微笑意、眉眼上扬、短促惊喜感、玩笑式挑眉、看见镜头后的自然反应、低头整理时的偷笑、看向环境时的兴奋感、被风吹到发丝后的短暂眯眼、试衣满意后的带笑抿唇、假装镇定后的轻笑等生活化表情。看镜头、侧脸、低头看手部动作、看向环境、看向镜中手机、回到自然表情等都可作为当次创作选择；表情变化服务卖点展示动作、S 型身材展示和才艺表现，不把镜头强行拉成纯脸部特写。结尾不强制看向镜头，也不为了封面单独设计固定停顿。
- 行程地点不是 prompt 场景的硬锁，具体画面可延展到酒店、机场、游泳馆、网球馆、商场、咖啡馆、餐厅、街区、酒吧、夜店等与当天行程合理相关的场景。
- prompt 只写可见画面、动作和镜头语言，不写合规说明；发布文案必须保持平台可发布表达。严禁生成低俗、裸体及未成年感内容。

## 内容边界

- 只写可见画面、动作、镜头、场景、穿搭和画面质感。
- 严禁低俗、裸体、未成年感和不可发布内容。
- 只执行 `anna auto/fast` 单图发布链路。

## 默认 auto/fast 流程

1. 建档：创建 `TEMP/RUN_ID/` 并初始化运行记录；默认不执行模块 00，只有遇到环境问题或用户明确要求修环境时才读取 `MODULE_00_ENV_REPAIR.md`。
2. 行程窗口：读取当前 active 的 `MATERIAL/anna-weekly-itinerary.json` 并按当天日期匹配行程；资产缺失、无效、当天日期早于 `valid_from` 或已超过 `valid_to` 时，才按今天起连续 7 天重新规划国内真实城市行程，直接保存到 `MATERIAL/` 后继续。
3. 自主选题与创作设计：按当天行程大方向先随机确定 `早上`、`上午`、`下午` 或 `晚上` 拍摄时段，再确定拍摄语境或展示舞台，并由执行者以短视频创作者视角设计作品的卖点展示结构、画面方案、场景、动作、镜头、发布文案方向和互动方式；设计必须优先服务当前赛道的修身穿搭、腰线、上身辨识度、成熟曲线和成熟女性妩媚感，生活动作只在能强化卖点时使用。
4. 衣柜按日期编号优先与 grid-prompt 写作：根据当天日期优先选择 `MATERIAL/anna-wardrobe.md` 中对应编号的款式，例如 7 号优先 `夏-07`、19 号优先 `夏-19`；如当天编号缺失或明显不匹配本次创作设计、active 行程、场景、天气、展示舞台或动作需求，执行者可自主设计或改选衣服，并记录原因。最终只提取 `款式提示词：` 内容，不把编号、季节标签、环境标签、元数据或使用建议写入最终 prompt。写入十段式 `grid-prompt.txt`，它是 Dreamina v1 最终 vid prompt 的派生来源。
5. 视频提示词：用 `TOOLS/prompt_lint.py derive --mode fast` 从 `grid-prompt.txt` 生成 `vid-prompt-v1.txt` 后提交 Dreamina；prompt 使用 `@图1` 指代 `anna.png`，不得含第二张图片引用、文件名、流程说明、合规说明、平台解释或“吸收/根据某文件”的表达。
6. 视频生成：只上传 `MATERIAL/fixed-role/anna.png` 作为 `@图1` 后提交 Dreamina 视频。生成成功并下载正式 MP4 到 `OUTPUT/RUN_ID.mp4` 后，执行基础质检和三张代理图人工视觉质检；默认全部通过即继续发布。只有用户明确要求“发布前确认”“只生成不发布”“本次不用发布”等暂停发布时，才展示视频、首中尾帧、vid prompt、TNS 记录和是否建议发布并等待授权。
7. 发布：通过模块 03 同步上传抖音和快手，两个平台都设置 `内容由AI生成` 声明；两个平台都返回 `published` 才判定整体发布成功，不要求发布后核对创作者中心内容列表。
8. 记录收尾：成功生成正式视频后记录产物路径和关键执行结果；发布后在运行记录中补充抖音、快手和整体发布状态并刷新记录；不把本次运行状态作为下次默认续跑依据。

## 硬阻断

- 进入自主选题、提示词、生成和发布前，必须具备有效的一周行程资产；资产缺失、无效、当天日期早于 `valid_from` 或已超过 `valid_to` 时，先重新生成今天起连续 7 天行程并保存为 active。
- 进入视频生成前，必须具备 `MATERIAL/fixed-role/anna.png`、十段式 `grid-prompt.txt` 和 `vid-prompt-v1.txt`。
- 视频生成输入合同为 `@图1` 单图；vid prompt 和 Dreamina 命令必须保持同一个图片输入。
- 视频生成因 TNS/安全拦截到 `v5` 仍未产出时停止，不发布，并报告 `v1-v5` 失败摘要。
- TNS 收敛只适用于生成工具明确返回安全拦截且没有产物；网络、登录、积分、参数、上传或超时等失败直接停止报告。
- CDP、Playwright、登录态、API key、积分、权限、素材、读写或网络代理等环境类失败时，先按模块 00 修复并记录可复用最佳实践。
- 默认自主分支生成正式 MP4，并通过基础质检和三张代理图人工视觉质检后直接进入模块 03；用户要求发布前确认时，收到授权后进入模块 03。
- 任一平台发布前必须完成 `内容由AI生成` 声明。
- 单个平台出现登录失效、验证码、账号安全、平台风控、上传失败、发布按钮禁用等阻断时，记录该平台失败并继续尝试另一个平台；两个平台执行完后报告整体状态。
