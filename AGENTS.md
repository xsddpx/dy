# dy auto 项目启动协议

## 启动读取

- 默认读取本文件和 `DOCS/PROJECT.md`，确认本项目默认执行 `anna auto/fast` 日更发布流程。
- 需要阶段细节时，按 `DOCS/PROJECT.md` 的固定流程读取 `DOCS/MODULES/` 下对应模块文档。
- 读取 `/Users/Shared/codex/Obsidian Vault/Askill/🎭AGENT须知.md`。

## 固定边界

- 本项目路径固定为 `/Users/Shared/codex/dy`。
- 只保留 `anna` 单人路线；默认模式为 `auto/fast`。
- 显式 `slow` 用于 Kie 确认图流程。
- `auto` 就是 `fast`，表示：参考抽帧 -> grid prompt -> 直接用 `anna.png` 单图提交 Dreamina -> 抖音发布 -> 记录收尾；不执行中间生图和选择节点。
- 默认复用当前 active 的 `MATERIAL/anna-weekly-itinerary.json`；只有资产缺失、无效、当天日期早于 `valid_from` 或已超过 `valid_to` 时，才重新生成今天起连续 7 天的国内真实城市行程并保存为 active，无需用户确认。
- 生成或重建一周行程资产时，每天不要写具体钟点；`time_slot` 统一写 `白天；晚上`，并为每天同时写清白天和晚上两个可拍活动、场景、穿搭方向和参考关键词。
- 行程资产和参考选择细节以 `DOCS/MODULES/MODULE_01_REFERENCE.md` 为准；行程地点只是当天主题大方向，不是具体参考地点或 prompt 环境的硬锁。
- `fast` 兼容触发词包括“快速通道”“跳过确认图”；这些都等同于 `auto/fast`，不再表示单独分支。
- `slow` 只有用户明确说 `slow`、`慢速模式`、`Kie 确认图`、`确认图流程` 或 `完整确认图流程` 时才启用，表示：参考抽帧 -> 提示词 -> Kie GPT Image 2 单张确认图 -> 确认图硬停确认 -> Dreamina 单图视频生成 -> 抖音发布 -> 记录收尾。
- 项目的关键节点按模式区分：`fast` 只有视频生成之后的视频确认关键节点；`slow` 包含确认图生成后确认和视频生成后确认两个关键节点。用户说“关键节点”时，不得自行扩展到参考选择、提示词、发布前等其他流程节点。
- `slow` 的确认图关键节点必须硬停：Kie 单张确认图生成后，必须向用户展示确认图、输入来源、img prompt、TNS 收敛记录和是否建议使用，并等待用户明确确认是否使用该图；建议使用不等于用户确认，未获确认不得写入最终选图记录、不得进入 Dreamina 视频生成。
- `slow` 确认图按批次单槽位生成：首批为 `A-01`；用户说“换一个”“换一张”或明确拒绝当前确认图时，不进入 Dreamina，也不停止整个 slow 任务，默认在同一 `RUN_ID`、同一参考和同一 `grid-prompt.txt` 下生成下一批单槽位，如 `B-01`、`C-01`；每批仍只允许一个 `*-01` 槽位。只有用户明确要求“并发 3 张”“一次生成 3 张”等并发换图时，才记录当前槽位 rejected，并在同一 `RUN_ID`、同一参考和同一 `grid-prompt.txt` 下并发生成后续三个单槽位批次，例如 `F-01`、`G-01`、`H-01`。
- `slow` 的视频确认关键节点必须硬停：Dreamina 生成 MP4 后，必须向用户展示视频、首中尾帧、vid prompt、TNS 收敛记录和是否建议发布，并等待用户明确确认发布；未获确认不得运行发布 helper、不得上传或发布到抖音。
- `grid-prompt.txt` 是 fast 的 Dreamina v1 最终 vid prompt，必须使用 `人物：`、`视频类型：`、`穿搭：`、`姿态镜头：`、`环境：`、`卖点与锁定：`、`表情节奏：`、`整体动画：`、`背景音乐：`、`其他：` 十段式；不保留分析字段、文件/流程来源、合规说明、平台解释、省略号、条件分支、模板说明或“根据/融合/吸收”等内容。
- `slow` 的确认图 img prompt 由同一份十段式 `grid-prompt.txt` 删除 `整体动画：` 和 `背景音乐：` 两段得到；特殊或不对称穿搭必须在 `穿搭：` 段锁定左右结构、袖长、领口方向、开口位置、裤腿或裙摆连续性和裁切范围。
- `slow` 的视频 vid prompt 从同一份十段式 `grid-prompt.txt` 派生，只删除人物段中的 anna 多视角角色卡声明，其他内容保持一致。
- 只执行上述 `anna auto/fast` 默认模式或显式 `slow` 模式，不接入其他生成路线、角色或兜底工具；不因 `auto/fast` 失败自动切换到 `slow`，也不因 `slow` 失败自动切回 `auto/fast`。
- TNS/安全拦截导致图片或视频未生成时，允许在当前模式、当前生成节点内从 `v1` 收敛到 `v5`：`v1` 是首次提交，`v2-v5` 是最多 4 次逐步收敛；到 `v5` 仍未生成则停止并报告，不切换模式、不接入其他路线或兜底工具。
- TNS 收敛只适用于生成工具明确返回 TNS/安全拦截且没有生成产物的情况；网络、登录、积分、参数错误、上传失败、超时等仍按硬阻断处理。
- 每次会话和每次自动化运行默认视为独立新任务；除非用户明确说继续/恢复并指定任务或产物，不读取或续接旧任务状态。
- 不因 `TEMP/` 或 `OUTPUT/` 中存在候选产物而默认发布或续跑旧任务。
