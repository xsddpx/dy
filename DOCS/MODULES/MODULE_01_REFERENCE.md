# 模块 01：参考、去重与宫格

## 职责

- 接收用户指定参考，或从抖音收藏抽样。
- 对自主参考执行 7 天去重。
- 生成 6 帧参考宫格和报告。
- 根据参考宫格或帧图反推可见导演结构，写入 `grid-prompt.txt`。

## grid-prompt 写作规范

- 找到收藏参考视频后，本次创意以复刻可见导演结构为主：先反推参考视频的场景、穿搭版型、构图、动作顺序、手部和身体路径、镜头距离、角度、运镜、表情节奏、节拍点和封面停顿；正式生成尽可能与参考视频一致。
- 默认只把人物身份替换为 `anna`；不复刻原视频人物身份、真人脸、账号标识、字幕水印、品牌商标或可能侵权的专有 IP。
- 正式 prompt 只写可见画面、动作、镜头、场景、穿搭和画面质感，不写合规说明。
- `grid-prompt.txt` 必须使用结构化记录，至少包含 `可见导演结构：`、`整体动画判断：`、`身材卖点校准：`、`参考六锁定：` 四部分，便于模块 03 人工重写最终 `vid prompt`。
- 需要有能体现性感穿搭模特身份和身材优势的穿搭描述：修身、肩颈线清晰、领口与上身轮廓清晰、腰线明确、面料张力、成熟曲线、轻熟性感；同时必须加入两组核心卖点的艺术化转译，不写直白词：
  - “胸部大”固定转译为：`饱满的立体廓形`、`高感知度的面料张力`、`上身丰盈的沙漏型线条`、`领口与上身轮廓清晰`。
  - “屁股大/臀部明显”固定转译为：`腰胯比例明显`、`臀胯轮廓自然凸显`、`高腰线与下装包裹出稳定曲线`、`古典雕塑般的 S 形动态`。
- 每次进入提示词前必须做身材卖点校准：成熟曲线、清晰腰线、上身动态辨识度、领口与上身轮廓、上胸线可读、修身面料张力、腰胯比例必须能被镜头读到。内部分析可直接记录大胸、乳沟、诱惑等增长判断；写入提示词时统一转译为具体可见的画面表达，如低领、方领、吊带、修身上衣、上胸线、肩颈线条、腰线、面料贴合和完整着装。抖音文案仍需保持平台可发布表达。弱卖点方向不得进入生成或发布。
- `身材卖点校准：` 必须明确写出肩颈线、领口与上身轮廓、修身面料张力、清晰腰线、腰胯比例、上身动态辨识度是否可被镜头读到，并给出进入 `vid prompt` 的可见画面表达。
- `grid-prompt.txt` 必须完成参考六锁定，并把结论写入正式记录；六锁定必须具体到可见姿态、镜头、场景、穿搭、身材描述、动作与封面停顿，禁止只写“性感”“好身材”“动作自然”等泛化审美词：
  - 姿态锁定：站姿、坐姿、半躺、侧身、前倾、重心、身体朝向、手臂和手部停留位置。
  - 镜头锁定：俯拍、平拍、仰拍、近景、超近景、半身、全身、裁切位置、固定机位或手持、跟拍。
  - 场景锁定：房间类型、背景物、家具、桌面、沙发、镜子、门窗、墙面、光线和人物与道具的空间关系。
  - 穿搭锁定：衣服类别、颜色、领口、肩臂露出、上衣长度、腰线位置、裙裤形态、面料贴合和外搭层次。
  - 身材描述锁定：参考里实际可见的上身视觉体量、肩颈线、上胸线、腰线、腰胯比例、腿部入镜范围、曲线来源和被衣服、姿态、镜头放大的部位。
  - 动作与封面锁定：开场姿态、动作路径、手部路径、节奏、最后停顿和最适合做封面的瞬间。

## 通过标准

- `reference-grid.jpg`、`reference-grid-report.json`、`reference-grid-report.md` 已写入 `TEMP/RUN_ID/`。
- `grid-prompt.txt` 已写入 `TEMP/RUN_ID/`，内容包括可见导演结构、人物动作连续性、镜头运动、节奏、姿态变化、画面动态重点、身材卖点校准和参考六锁定结论。
- 宫格报告 `decision=pass` 且 `validation.decision=pass`。
- 宫格用于 img prompt 和 grid prompt 分析与记录，不作为 Dreamina 视频生成的视觉输入，也不得直接作为 Kie 确认图输入。
