# 模块 01：自主选题、衣柜匹配与 grid prompt

## 职责

围绕推荐卖点自主设计作品，从 Anna 衣柜优先匹配款式，将胜出的八段式 prompt 写入 `TEMP/RUN_ID/grid-prompt.txt`，再通过 `TOOLS/prompt_lint.py derive` 派生视频 prompt。

## 自主创作流程

1. 完成五项判断：本条视频卖什么、主镜头关系是什么、一个主要动作目标是什么、穿搭如何突出身材、场景如何服务核心卖点。
2. 从 `早上`、`上午`、`下午`、`晚上` 中随机确定拍摄时段；用户指定时段时优先遵循。城市、季节、光线和空间细节由本次创作自主确定并保持一致。
3. 生成三条差异清楚的候选方向，差异落在镜头关系、主要动作目标、场景舞台、才艺表现或卖点角度；至少两条使用室内场景，胜出方案默认优先室内。用户指定室外或室内确实不适配时可选择室外并记录原因。
4. 镜头关系选择 `固定拍摄` 或 `他人手持拍摄` 并贯穿始终。固定拍摄保持机位不变；他人手持保持稳定构图，可围绕人物小幅调整角度。拍摄设备始终位于画外。
5. 每条只确定一个主要动作目标，例如松弛轻舞展示、自然转身互动、短距离走近回看或道具互动；人物保持松弛妩媚和连续流动感，prompt 省略具体手脚路径、身体衔接、视线、表情、快慢节奏和固定结尾。人物全程留在画面内，移动范围以镜头能够稳定承接为准。
6. 新鲜点可来自构图、景别、主要动作目标、才艺或道具互动；最近三条使用不同的空间与动作方向，不为追求差异堆叠分解动作。
7. 完成创作设计后，按当天日期优先选择 `MATERIAL/anna-wardrobe.md` 对应编号。款式缺失或不适配卖点、场景和动作时可自主改选，并记录原因；写入 `穿搭：` 时只提取 `款式提示词：` 正文。
8. 按卖点可见性、镜头一致性、动作目标清晰度、生成自由度、新鲜度、TNS 风险和八段式可落地性选出一条；背景协调度只作低权重参考。候选只用于内部筛选，`grid-prompt.txt` 只写胜出结果。

## grid-prompt 固定结构

八个标签各出现一次并按以下顺序排列，每段冒号后直接写最终可执行内容：

- `人物：`
- `视频类型：`
- `穿搭：`
- `姿态镜头：`
- `环境：`
- `整体动画：`
- `背景音乐：`
- `其他：`

`人物：` 固定使用以下身份与身材锚点：

```text
人物：@图1 是同一位成年女性的多视角、多表情角色参考图，不是多人合照；以 @图1 中左下大脸和正面脸为主要身份依据，特别保持五官比例、脸型、眼神、鼻唇结构、发型分线和自然神态；以 @图1 中正面、侧面和背面全身图为稳定身材依据，特别保持上身体量、胸部体量比例、纤细腰线、腰胯比例和整体 S 型曲线。侧面、背面和表情小图只用于辅助保持发型、脸型、身材比例和整体气质。画面中只出现这一位成年女性。
```

## 视频类型

- `视频类型：` 写一个主类型、最多一个次类型和一句短的成片方向；不需要次类型时写 `次类型：无`。
- 类型集合固定为：`舞蹈律动`、`健身运动`、`穿搭展示`、`走路回头`、`坐姿互动`、`双人互动`、`生活场景剧情`。
- `双人互动` 仅为历史兼容类型名，当前路线固定呈现 Anna 单人；`走路回头` 只在主题确实需要一步内出门动作时使用。

## 段落写作规则

- `人物：` 使用固定锚点；穿搭、姿态和环境分别写入对应段落。
- `穿搭：` 写清类别、颜色、领口、左右肩臂、袖长、上衣长度、腰线、裙裤形态、面料贴合、外搭层次和裁切范围。不对称款式进一步明确左右结构、开口或露腿位置及裁切。
- `姿态镜头：` 开头明确 `固定拍摄` 或 `他人手持拍摄`，写清景别、裁切、角度、一个主要动作目标和可见卖点。只规定人物要完成什么和镜头要看到什么，不编排手脚、肩胯、视线与表情的先后路径。近景聚焦表情、肩颈、领口和上身轮廓；中近景聚焦穿搭轮廓、面料张力和腰线；中景或全身呈现动作与整体比例。
- `环境：` 写清空间、背景物、材质、光线、纵深和人物关系，优先服务卖点、动作与构图，并保持时段、城市和季节逻辑一致。
- `整体动画：` 用简短目标式语言描述连续动态方向、松弛妩媚的人物状态和期望的画面结果。时长由 Dreamina 参数控制；不写逐秒时间线、身体部位操作顺序、精确手脚路径、固定结尾姿势，以及“生成 AI”“自由发挥”“自主完成”等不可见的生成说明。
- `背景音乐：` 写抖音热门的音乐风格、节奏和情绪，保持纯音乐表达。
- `其他：` 使用正向摄影约束：`写实摄影风格，真实人物质感，自然光影，真实皮肤纹理，真实面部结构，真实头发丝细节，真实服装材质，符合物理规律的光照和阴影，自然景深，真实镜头质感，真实环境透视，真实色彩。` 同时明确穿搭轮廓、腰线可读、构图稳定、单一连续完整竖屏画面和物理真实，排除拼图、分屏、多格和多姿势拼贴。
- 最终正文只写可见画面语言，不保留占位符、条件分支、内部分析、文件来源、流程说明、合规说明或平台解释。

## 推荐卖点转译

本节是建议，不是强制锁定；按景别、动作连贯性、成片观感和平台安全感灵活取舍，并写进 `姿态镜头：`。

- 优先让 S 型成熟曲线、清晰腰线、上身动态辨识度、领口与上身轮廓、修身面料张力和腰胯比例被镜头读到。
- 胸部体量可转译为：`饱满的立体廓形`、`高感知度的面料张力`、`上身丰盈的沙漏型线条`、`领口与上身轮廓清晰`。
- 臀胯比例可转译为：`腰胯比例明显`、`臀胯轮廓自然凸显`、`高腰线与下装包裹出稳定曲线`、`古典雕塑般的 S 形动态`。

## 派生与通过标准

```bash
python3 TOOLS/prompt_lint.py derive TEMP/RUN_ID/grid-prompt.txt --mode fast --out TEMP/RUN_ID/vid-prompt-v1.txt
```

- 已完成五项判断、三条差异候选、室内候选数量和日期衣柜检查；改用室外或其他衣柜款式时已记录原因。
- 胜出 prompt 像真人认真拍摄的短视频，保持一个镜头关系和一个清楚的主要动作目标；人物状态与画面结果明确，省略动作分解与固定结尾，并与最近三条保持差异。
- 八段标签、人物锚点、穿搭结构、可见卖点、环境逻辑和动作镜头一致性已通过检查。
- `grid-prompt.txt` 只含一条胜出结果；`vid-prompt-v1.txt` 由 `derive --mode fast` 生成并通过校验，派生只标准化文件末尾换行。
