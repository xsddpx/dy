# 模块 01：参考、去重与 grid prompt

## 职责

- 接收用户指定参考，或从抖音收藏抽样。
- 对自主参考执行 7 天去重。
- 生成 6 帧参考宫格和报告。
- 执行者直接查看 `reference-grid.jpg` 或帧图，反推可见导演结构。
- 写入 `TEMP/RUN_ID/grid-prompt.txt`，且这份文件就是 Dreamina v1 的最终 vid prompt。

## 核心原则

- 找到收藏参考视频后，本次创意以复刻可见导演结构为主：反推场景、穿搭版型、构图、动作顺序、手部和身体路径、镜头距离、角度、运镜、表情节奏、节拍点和封面停顿；正式生成尽可能与参考视频一致。
- 默认只把人物身份替换为 `anna`；不复刻原视频人物身份、真人脸、账号标识、字幕水印、品牌商标或可能侵权的专有 IP。
- 参考宫格和帧图只用于人工视觉分析，不作为 Kie 或 Dreamina 的视觉输入。
- 视频类型判断、身材卖点校准、参考锁定和取舍只用于执行者写作前的内部判断；这些判断不得作为分析字段写进 `grid-prompt.txt`。
- `grid-prompt.txt` 只写可直接提交 Dreamina 的画面、动作、镜头、场景、穿搭、表情、音乐和质感描述，不写文件名、来源说明、流程说明、合规说明、平台解释或“根据/融合/吸收”等解释性表达。
- 不新增脚本拼接 prompt；执行者必须根据宫格或帧图人工写成一份连贯、无矛盾、可直接提交的最终文本。

## grid-prompt 固定结构

`grid-prompt.txt` 必须使用十段式结构，且段落顺序固定：

- `人物：`
- `视频类型：`
- `穿搭：`
- `姿态镜头：`
- `环境：`
- `卖点与锁定：`
- `表情节奏：`
- `整体动画：`
- `背景音乐：`
- `其他：`

每段冒号后都必须写最终可执行内容，不得保留省略号、尖括号占位、条件分支或模板说明。`人物：` 段按当前模式二选一写成确定句。

`auto/fast` 使用：

```text
人物：以 @图1 中的同一位成年女性作为身份、五官、脸型、发型、自然神态和稳定身材比例依据，保持同一人物的一致性。画面中只出现这一位成年女性。
```

`slow` 使用：

```text
人物：以 @图1 中唯一成年女性作为身份、五官、发型、脸型、自然神态、稳定身材比例和画面锚点。画面中只出现这一位成年女性。
```

## 视频类型规则

- `视频类型：` 段必须写主类型、次类型和一句短的成片方向，例如 `视频类型：穿搭展示；次类型：健身运动；近景穿搭展示结合轻运动互动节奏。`。
- 主类型只能选择一个最贴近的类型：`舞蹈律动`、`健身运动`、`穿搭展示`、`镜前自拍`、`走路回头`、`坐姿互动`、`双人互动`、`生活场景剧情`。
- 次类型可选且最多一个；需要时从同一集合中选择，不需要时写 `次类型：无`。不得新增其他类型名。
- 类型判断提示：
  - `舞蹈律动`：音乐节拍驱动，身体、手臂、肩颈或髋部有连续律动，动作重点是节奏而非展示某件衣服。
  - `健身运动`：动作来自训练、拉伸、瑜伽、运动装备展示或健身房/运动空间，身体活动和运动状态是主线。
  - `穿搭展示`：镜头主要展示服装版型、上身轮廓、腰线、整体搭配和封面姿态，动作多为站姿、转身、整理衣服或轻互动。
  - `镜前自拍`：镜子、手机、自拍视角或镜中人物关系是主要构图来源。
  - `走路回头`：人物行走、靠近/远离、回头、转身或跟拍路径是主要动作线。
  - `坐姿互动`：人物坐在沙发、椅子、床边、车内等位置，动作以坐姿调整、手势、眼神互动为主。
  - `双人互动`：参考里有两人或多人互动；dy 仍只执行 anna 单人路线，后续必须把互动关系转译为单人可执行动作，不启用双人路线。
  - `生活场景剧情`：动作有清晰日常事件或情节目标，例如进门、拿东西、整理房间、做饭、通勤准备。

## 段落写作规则

- `人物：` 使用当前模式的确定提示词，只负责身份、五官、发型、脸型、神态和稳定身材比例；`auto/fast` 和 slow 确认图不得把 `anna.png` 的穿搭当作依据，穿搭、动作和场景必须来自本次参考分析。
- `视频类型：` 写主类型、次类型和一句短的成片方向，不写内部判断依据。
- `穿搭：` 必须直接来自宫格或帧图的可见画面分析，先锁定服装结构，再写材质与贴合；必须写清衣服类别、颜色、领口方向、左右肩臂露出关系、袖长、上衣长度、腰线、裙裤形态、面料贴合、外搭层次和画面裁切范围。
- 特殊或不对称穿搭必须写清左右结构：哪一侧露肩、哪一侧长袖、斜向领口从哪侧延伸到哪侧、腰线位置、下装左右差异、开口或露腿位置、裤腿或裙摆是否连续、袖口或裤腿到画面何处被裁切。
- 宫格中能看到但容易被模型改写的部位必须显式描述，例如袖口到手腕、单侧高开线条、另一侧长裤腿、胯部与大腿上部裁切、肩带或领口是否连续；参考不清晰时只写可确认部分，不脑补品牌、配饰或不可见层次。
- `姿态镜头：` 必须写清站坐姿、侧身或前倾、重心、身体朝向、手部位置、裁切、镜头距离、角度、固定或手持关系、运镜关系和封面姿态。
- `环境：` 必须写清空间类型、背景物、家具或道具、材质、光线来源、空间纵深和人物与环境关系。
- `卖点与锁定：` 必须写入穿搭版型、领口与上身轮廓、面料张力、腰线、腰胯比例、姿态、镜头裁切、动作重点和封面停顿；禁止只写“性感”“好身材”“动作自然”等泛化审美词。
- `表情节奏：` 必须参考可见画面，拆成开场、中段、结尾的眼神、嘴角、眉眼和脸部状态；参考不清晰时写自然平静、专注或放松，不主动脑补成微笑。微笑可以记录，但不得把“最后微笑”当作默认收束。
- `整体动画：` 围绕约 5-6 秒成片重新编排连续动作、手部路径、身体重心变化、镜头距离、角度、运镜、节奏点和结尾停顿，不按参考原时长逐秒照搬。
- `背景音乐：` 只写音乐风格、节奏和情绪，不写歌词、对白、口播、环境声、脚步声、衣料声、镜头声、喘息或音效。
- `其他：` 只写真实皮肤纹理、自然光影、真实面料质感、穿搭轮廓、腰线可读、构图稳定和画面物理真实；不写排除清单、合规说明或平台解释。

## 卖点表达规则

- 每次进入提示词前必须做身材卖点校准：成熟曲线、清晰腰线、上身动态辨识度、领口与上身轮廓、上胸线可读、修身面料张力、腰胯比例必须能被镜头读到；弱卖点方向不得进入生成或发布。
- 核心卖点只用可见画面语言表达，不写直白词：
  - “胸部大”固定转译为：`饱满的立体廓形`、`高感知度的面料张力`、`上身丰盈的沙漏型线条`、`领口与上身轮廓清晰`。
  - “屁股大/臀部明显”固定转译为：`腰胯比例明显`、`臀胯轮廓自然凸显`、`高腰线与下装包裹出稳定曲线`、`古典雕塑般的 S 形动态`。
- `卖点与锁定：` 必须覆盖姿态、镜头、场景、穿搭、身材描述、动作与封面停顿六项最终取舍，但只写成可执行画面语言，不写内部条目标题。

## img prompt 派生规则

- slow 的确认图 img prompt 从同一份 `grid-prompt.txt` 派生，只删除 `整体动画：` 和 `背景音乐：` 两段。
- img prompt 保留 `人物：`、`视频类型：`、`穿搭：`、`姿态镜头：`、`环境：`、`卖点与锁定：`、`表情节奏：`、`其他：` 八段。
- 派生后的 img prompt 仍必须是 Kie 可直接执行的静态画面描述；如果删除动画和音乐后出现断裂、矛盾或动态动作过强，必须回到 `grid-prompt.txt` 重写对应段落。


## 通过标准

- `reference-grid.jpg`、`reference-grid-report.json`、`reference-grid-report.md` 已写入 `TEMP/RUN_ID/`。
- `grid-prompt.txt` 已写入 `TEMP/RUN_ID/`，且包含十段式最终 vid prompt。
- 宫格报告 `decision=pass` 且 `validation.decision=pass`。
- `grid-prompt.txt` 可逐字复制为 `vid-prompt-v1.txt` 并直接提交 Dreamina。
- slow 的 img prompt 可由同一份 `grid-prompt.txt` 删除动画和音乐两段得到。
- 宫格用于 prompt 分析与记录，不作为 Kie 或 Dreamina 的视觉输入。
