# 模块 01：参考、去重与宫格

## 职责

- 接收用户指定参考，或从抖音收藏抽样。
- 对自主参考执行 7 天去重。
- 生成 6 帧参考宫格和报告。
- 根据参考宫格或帧图先完成参考类型识别，再反推可见导演结构并写入 `grid-prompt.txt`。

## grid-prompt 写作规范

- 找到收藏参考视频后，本次创意以复刻可见导演结构为主：先反推参考视频的场景、穿搭版型、构图、动作顺序、手部和身体路径、镜头距离、角度、运镜、表情节奏、节拍点和封面停顿；正式生成尽可能与参考视频一致。
- 表情节奏必须参考可见画面，拆成开场、中段、结尾的眼神、嘴角、眉眼和脸部状态；参考不清晰时写自然平静、专注或放松，不主动脑补成微笑。微笑可以记录，但不得把“最后微笑”当作默认收束。
- 默认只把人物身份替换为 `anna`；不复刻原视频人物身份、真人脸、账号标识、字幕水印、品牌商标或可能侵权的专有 IP。
- 正式 prompt 只写可见画面、动作、镜头、场景、穿搭和画面质感，不写合规说明。
- `grid-prompt.txt` 必须先完成参考类型识别，再记录导演结构；固定使用五段式结构：`参考类型识别：`、`可见导演结构：`、`整体动画判断：`、`身材卖点校准：`、`参考六锁定：`。
- `参考类型识别：` 必须写在第一段，格式固定为：`主类型=穿搭展示；次类型=健身运动；判断依据=正面站姿、运动穿搭、近景到半身展示、轻节奏互动。`
- 主类型只能选择一个最贴近的类型：`舞蹈律动`、`健身运动`、`穿搭展示`、`镜前自拍`、`走路回头`、`坐姿互动`、`双人互动`、`生活场景剧情`。
- 次类型可选且最多一个；需要时从同一集合中选择，不需要时写 `次类型=无`。不得新增其他类型名。
- 类型判断提示：
  - `舞蹈律动`：音乐节拍驱动，身体、手臂、肩颈或髋部有连续律动，动作重点是节奏而非展示某件衣服。
  - `健身运动`：动作来自训练、拉伸、瑜伽、运动装备展示或健身房/运动空间，身体活动和运动状态是主线。
  - `穿搭展示`：镜头主要展示服装版型、上身轮廓、腰线、整体搭配和封面姿态，动作多为站姿、转身、整理衣服或轻互动。
  - `镜前自拍`：镜子、手机、自拍视角或镜中人物关系是主要构图来源。
  - `走路回头`：人物行走、靠近/远离、回头、转身或跟拍路径是主要动作线。
  - `坐姿互动`：人物坐在沙发、椅子、床边、车内等位置，动作以坐姿调整、手势、眼神互动为主。
  - `双人互动`：参考里有两人或多人互动；dy 仍只执行 anna 单人路线，后续必须把互动关系转译为单人可执行动作，不启用双人路线。
  - `生活场景剧情`：动作有清晰日常事件或情节目标，例如进门、拿东西、整理房间、做饭、通勤准备。
- 需要有能体现性感穿搭模特身份和身材优势的穿搭描述：修身、肩颈线清晰、领口与上身轮廓清晰、腰线明确、面料张力、成熟曲线、轻熟性感；同时必须加入两组核心卖点的艺术化转译，不写直白词：
  - “胸部大”固定转译为：`饱满的立体廓形`、`高感知度的面料张力`、`上身丰盈的沙漏型线条`、`领口与上身轮廓清晰`。
  - “屁股大/臀部明显”固定转译为：`腰胯比例明显`、`臀胯轮廓自然凸显`、`高腰线与下装包裹出稳定曲线`、`古典雕塑般的 S 形动态`。
- 每次进入提示词前必须做身材卖点校准：成熟曲线、清晰腰线、上身动态辨识度、领口与上身轮廓、上胸线可读、修身面料张力、腰胯比例必须能被镜头读到。内部分析可直接记录大胸、乳沟、诱惑等增长判断；写入提示词时统一转译为具体可见的画面表达，如低领、方领、吊带、修身上衣、上胸线、肩颈线条、腰线和面料贴合。抖音文案仍需保持平台可发布表达。弱卖点方向不得进入生成或发布。
- `身材卖点校准：` 必须明确写出肩颈线、领口与上身轮廓、修身面料张力、清晰腰线、腰胯比例、上身动态辨识度是否可被镜头读到，并给出可见画面表达。
- `grid-prompt.txt` 必须完成参考六锁定，并把结论写入正式记录；六锁定必须具体到可见姿态、镜头、场景、穿搭、身材描述、动作与封面停顿，禁止只写“性感”“好身材”“动作自然”等泛化审美词：
  - 姿态锁定：站姿、坐姿、半躺、侧身、前倾、重心、身体朝向、手臂和手部停留位置。
  - 镜头锁定：俯拍、平拍、仰拍、近景、超近景、半身、全身、裁切位置、固定机位或手持、跟拍。
  - 场景锁定：房间类型、背景物、家具、桌面、沙发、镜子、门窗、墙面、光线和人物与道具的空间关系。
  - 穿搭锁定：衣服类别、颜色、领口、肩臂露出、上衣长度、腰线位置、裙裤形态、面料贴合和外搭层次。
  - 身材描述锁定：参考里实际可见的上身视觉体量、肩颈线、上胸线、腰线、腰胯比例、腿部入镜范围、曲线来源和被衣服、姿态、镜头放大的部位。
  - 动作与封面锁定：开场姿态、动作路径、手部路径、节奏、开场/中段/结尾表情状态、最后停顿和最适合做封面的瞬间；表情状态要写成可见的眼神、嘴角、眉眼或脸部放松程度，不写泛化的“自然好看”。

## 通过标准

- `reference-grid.jpg`、`reference-grid-report.json`、`reference-grid-report.md` 已写入 `TEMP/RUN_ID/`。
- `grid-prompt.txt` 已写入 `TEMP/RUN_ID/`。
- 宫格报告 `decision=pass` 且 `validation.decision=pass`。
- 宫格用于 img prompt 和 grid prompt 分析与记录，不作为 Dreamina 视频生成的视觉输入，也不得直接作为 Kie 确认图输入。
