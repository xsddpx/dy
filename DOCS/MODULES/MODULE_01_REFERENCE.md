# 模块 01：参考、去重与 grid prompt

## 职责

- 接收用户指定参考，或从抖音收藏抽样。
- 对自主参考执行 7 天去重。
- 生成 6 帧参考宫格和报告。
- 根据参考宫格或帧图先完成参考类型识别，再写入统一 `grid-prompt.txt`。
- `grid-prompt.txt` 是后续 img prompt 和 vid prompt 的统一规则库与可复用拼装块。

## 核心原则

- 找到收藏参考视频后，本次创意以复刻可见导演结构为主：反推场景、穿搭版型、构图、动作顺序、手部和身体路径、镜头距离、角度、运镜、表情节奏、节拍点和封面停顿；正式生成尽可能与参考视频一致。
- 默认只把人物身份替换为 `anna`；不复刻原视频人物身份、真人脸、账号标识、字幕水印、品牌商标或可能侵权的专有 IP。
- 参考宫格和帧图只用于人工视觉分析，不作为 Kie 或 Dreamina 的视觉输入。
- `grid-prompt.txt` 允许保留分析索引；最终 img/vid prompt 只写可直接执行的画面描述，不写文件名、流程说明、来源说明、分析过程或“根据/融合/吸收”等解释性表达。
- 后续 prompt 采用模块化人工拼装：从 `grid-prompt.txt` 取用标准块并做必要清理，不新增脚本自动拼接，也不得机械堆叠矛盾信息。

## grid-prompt 固定结构

`grid-prompt.txt` 必须使用下列九段式结构：

```text
参考类型识别：主类型=穿搭展示；次类型=健身运动；判断依据=正面站姿、运动穿搭、近景到半身展示、轻节奏互动。
人物：以 @图1 中的人物作为唯一身份、五官、发型、脸型、自然神态和稳定身材比例依据；若 @图1 是多视角角色参考图，以左下大脸和正面脸为主要身份依据，侧面、背面和表情小图只用于辅助保持发型、脸型、身材比例和整体气质；若 @图1 是已确认单图，以该图中唯一成年女性的身份、姿态和画面锚点为准。画面中只出现这一位成年女性。
穿搭：...
姿态镜头：...
环境：...
卖点与锁定：...
整体动画：视频类型为穿搭展示，次类型为健身运动；...
背景音乐：...
其他：真实皮肤纹理，自然光影，真实面料质感，穿搭轮廓清晰，腰线可见，构图稳定，画面物理真实；不出现原视频人物身份、真人脸、账号标识、字幕水印、品牌商标或专有 IP。
```

## 参考类型识别

- `参考类型识别：` 必须写在第一段，格式固定为：`主类型=穿搭展示；次类型=健身运动；判断依据=正面站姿、运动穿搭、近景到半身展示、轻节奏互动。`
- 主类型只能选择一个最贴近的类型：`舞蹈律动`、`健身运动`、`穿搭展示`、`镜前自拍`、`走路回头`、`坐姿互动`、`双人互动`、`生活场景剧情`。
- 次类型可选且最多一个；需要时从同一集合中选择，不需要时写 `次类型=无`。不得新增其他类型名。
- 类型判断提示：
  - `舞蹈律动`：音乐节拍驱动，身体、手臂、肩颈或髋部有连续律动，动作重点是节奏而非展示某件衣服。
  - `健身运动`：动作来自训练、拉伸、瑜伽、运动装备展示或健身房/运动空间，身体活动和运动状态是主线。
  - `穿搭展示`：镜头主要展示服装版型、上身轮廓、腰线、整体搭配和封面姿态，动作多为站姿、转身、整理衣服或轻互动。
  - `镜前自拍`：镜子、手机、自拍视角或镜中人物关系是主要构图来源。
  - `走路回头`：人物行走、靠近/远离、回头、转身或跟拍路径是主要动作线。
  - `坐姿互动`：人物坐在沙发、椅子、床边、车内等位置，动作以坐姿调整、手势、眼神互动为主。
  - `双人互动`：参考里有两人或多人互动；dy 仍只执行 anna 单人路线，后续必须把互动关系转译为单人可执行动作，不启用双人路线。
  - `生活场景剧情`：动作有清晰日常事件或情节目标，例如进门、拿东西、整理房间、做饭、通勤准备。

## 可复用段落规则

- `人物：` 使用固定提示词，只负责身份、五官、发型、脸型、神态和稳定身材比例；`auto/fast` 和 slow 确认图不得把 `anna.png` 的穿搭当作依据，穿搭、动作和场景必须来自本次参考分析。
- `穿搭：` 必须直接来自宫格或帧图的可见画面分析，先锁定服装结构，再写材质与贴合；必须写清衣服类别、颜色、领口方向、左右肩臂露出关系、袖长、上衣长度、腰线、裙裤形态、面料贴合、外搭层次和画面裁切范围。
- 特殊或不对称穿搭必须写清左右结构：哪一侧露肩、哪一侧长袖、斜向领口从哪侧延伸到哪侧、腰线位置、下装左右差异、开口或露腿位置、裤腿或裙摆是否连续、袖口或裤腿到画面何处被裁切。
- 宫格中能看到但容易被模型改写的部位必须显式描述，例如袖口到手腕、单侧高开线条、另一侧长裤腿、胯部与大腿上部裁切、肩带或领口是否连续；参考不清晰时只写可确认部分，不脑补品牌、配饰或不可见层次。
- `姿态镜头：` 必须写清站坐姿、侧身或前倾、重心、身体朝向、手部位置、裁切、镜头距离、角度、固定或手持关系、运镜关系和封面姿态。
- `环境：` 必须写清空间类型、背景物、家具或道具、材质、光线来源、空间纵深和人物与环境关系。
- `卖点与锁定：` 合并身材卖点校准、参考六锁定和最终 prompt 的卖点规则；必须写入穿搭版型、领口与上身轮廓、面料张力、腰线、腰胯比例、姿态、镜头裁切、动作重点和封面停顿。
- `卖点与锁定：` 必须覆盖六项锁定，禁止只写“性感”“好身材”“动作自然”等泛化审美词：
  - 姿态锁定：站姿、坐姿、半躺、侧身、前倾、重心、身体朝向、手臂和手部停留位置。
  - 镜头锁定：俯拍、平拍、仰拍、近景、超近景、半身、全身、裁切位置、固定机位或手持、跟拍。
  - 场景锁定：房间类型、背景物、家具、桌面、沙发、镜子、门窗、墙面、光线和人物与道具的空间关系。
  - 穿搭锁定：衣服类别、颜色、领口、肩臂露出、上衣长度、腰线位置、裙裤形态、面料贴合和外搭层次。
  - 身材描述锁定：参考里实际可见的上身视觉体量、肩颈线、上胸线、腰线、腰胯比例、腿部入镜范围、曲线来源和被衣服、姿态、镜头放大的部位。
  - 动作与封面锁定：开场姿态、动作路径、手部路径、节奏、开场/中段/结尾表情状态、最后停顿和最适合做封面的瞬间。
- 核心卖点只用可见画面语言表达，不写直白词：
  - “胸部大”固定转译为：`饱满的立体廓形`、`高感知度的面料张力`、`上身丰盈的沙漏型线条`、`领口与上身轮廓清晰`。
  - “屁股大/臀部明显”固定转译为：`腰胯比例明显`、`臀胯轮廓自然凸显`、`高腰线与下装包裹出稳定曲线`、`古典雕塑般的 S 形动态`。
- 每次进入提示词前必须做身材卖点校准：成熟曲线、清晰腰线、上身动态辨识度、领口与上身轮廓、上胸线可读、修身面料张力、腰胯比例必须能被镜头读到；弱卖点方向不得进入生成或发布。
- `整体动画：` 只用于 vid prompt，必须以 `视频类型为主类型，次类型为次类型；` 开头；次类型为无时写 `视频类型为主类型，次类型为无；`。随后围绕约 5-6 秒成片重新编排连续动作、手部路径、身体重心变化、镜头距离、角度、运镜、节奏点、参考表情节奏和结尾停顿，不按参考原时长逐秒照搬。
- 表情节奏必须参考可见画面，拆成开场、中段、结尾的眼神、嘴角、眉眼和脸部状态；参考不清晰时写自然平静、专注或放松，不主动脑补成微笑。微笑可以记录，但不得把“最后微笑”当作默认收束。
- `背景音乐：` 只用于 vid prompt，只写音乐风格、节奏和情绪，不写歌词、对白、口播、环境声、脚步声、衣料声、镜头声、喘息或音效。
- `其他：` 固定写真实皮肤纹理、自然光影、真实面料质感、构图稳定、画面物理真实，并排除原视频人物身份、真人脸、账号标识、字幕水印、品牌商标或专有 IP；vid prompt 还必须明确除背景音乐外不出现任何非音乐声音。

## 拼装规则

- img prompt 使用六段式：`人物：`、`穿搭：`、`姿态镜头：`、`环境：`、`卖点与锁定：`、`其他：`。
- vid prompt 使用八段式：`人物：`、`穿搭：`、`姿态镜头：`、`环境：`、`卖点与锁定：`、`整体动画：`、`背景音乐：`、`其他：`。
- `参考类型识别：` 只作分析索引，不复制到最终 img/vid prompt；`整体动画：` 和 `背景音乐：` 只进入 vid prompt。
- final img/vid prompt 不得出现 `grid-prompt.txt`、`reference-grid`、`参考宫格`、`参考类型识别`、`主类型=`、`次类型=`、`判断依据=`、`根据`、`融合`、`吸收`、`文件`、`流程`、`节点` 等内部来源词或解释性表达。
- 若 grid prompt 某个标准块缺失或互相矛盾，先回到模块 01 重写 `grid-prompt.txt`，再进入 Kie 或 Dreamina。

- `reference-grid.jpg`、`reference-grid-report.json`、`reference-grid-report.md` 已写入 `TEMP/RUN_ID/`。
- `grid-prompt.txt` 已写入 `TEMP/RUN_ID/`，且包含九段式结构。
- 宫格报告 `decision=pass` 且 `validation.decision=pass`。
- `grid-prompt.txt` 已能支持 img 六段式和 vid 八段式拼装。
- 宫格用于 prompt 分析与记录，不作为 Kie 或 Dreamina 的视觉输入。
