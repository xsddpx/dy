- 按 `DOCS/PROJECT.md` 的流程路由读取 `DOCS/MODULES/` 对应模块。

## 固定边界

- 本项目路径固定为 `/Users/Shared/codex/dy`。
- 只执行 `anna` 单人路线；默认 `auto/fast`，只有用户明确指定 slow 触发词时才进入 Kie 确认图流程，两种模式不自动切换。
- `fast` 默认在视频生成成功后直接同步发布到抖音和快手；只有用户明确要求视频确认、只生成不发布或发布前确认时才硬停。
- `slow` 固定保留两个硬停：确认图生成后等待选图，视频生成后等待确认发布。
- 模块 00 是环境修复模块，默认跳过；只有遇到 CDP、Playwright、登录态、API key、积分、权限、素材、读写、网络代理等环境问题，或用户明确要求修环境时才执行。
- 当前 active 一周行程在有效窗口内必须复用；缺失、无效、未开始或过期时按模块 01 重建后继续。
- Dreamina 视频只允许单图输入；fast 使用 `anna.png`，slow 使用用户确认的 Kie 原始图。参考宫格只用于分析。
- TNS 收敛仅用于生成工具明确返回安全拦截且没有产物的情况，最多到 `v5`；其他失败直接停止报告。
- 抖音和快手都必须尝试发布；两个平台都返回 `published` 才判定整体发布成功，不要求发布后核对创作者中心内容列表。
- 每次会话和每次自动化运行默认视为独立新任务；除非用户明确说继续/恢复并指定任务或产物，不读取或续接旧任务状态。
- 不因 `TEMP/` 或 `OUTPUT/` 中存在候选产物而默认发布或续跑旧任务。

## 文档边界

- `DOCS/PROJECT.md` 保存核心事实、流程路由和全局硬阻断。
- 环境修复和最佳实践沉淀只看模块 00；行程、参考和 prompt 规范只看模块 01；slow 确认图只看模块 02；视频输入与确认节点只看模块 03；发布只看模块 04。
- 修改核心事实或提示词规则前必须先说明拟修改内容和原因；用户明确要求修改时，本次授权仅限其明确范围。
