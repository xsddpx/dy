# 主链 02：内容规划与 Prompt

## 职责与单一事实源

本模块锁定衣柜、环境、动作、时长和随机 seed，并直接写出唯一正式 prompt：`TEMP/RUN_ID/vid-prompt-vN.txt`。不创建 `grid-prompt.txt`，也不存在 derive 步骤。

机器可执行事实只保存在 `MATERIAL/xdy-workflow.json`：模型、比例、分辨率、合法时长、双图资产、七段顺序、固定段落、动作模板及默认时长均从该配置读取。本文只解释选择与创作规则，不复制固定模板。配置与 prompt 均通过 SHA-256 锁入生成清单。

## 创作目标与硬边界

- Anna 始终是同一位明确成年的女性，完整穿着，写实、成熟、自然。
- 核心视觉是修身穿搭、清晰腰线、稳定上身辨识度、活泼互动与可见才艺。
- 固定角色图为 `MATERIAL/fixed-role/anna.png`，始终作为 `@图1`；本次正式环境图始终作为 `@图2`。
- 固定人物段必须显式锁定胸部体量：以角色图正面、斜侧面和侧面全身图为准，保持相同体量、胸廓前后比例和侧向投影，各角度全程稳定一致。该规则属于共享人物模板，`xdy` 与 `xdysp` 的每个 prompt 版本都必须保留。
- 七段严格按配置顺序出现一次；最终正文只写可见画面，不写占位符、条件分支、内部分析、文件来源、流程或平台解释。
- 视频约束必须含“全程保持正常速度，不使用慢动作”，并由配置校验和回归测试共同保护。

## 近期去重轮换

用户指定的有效衣柜、环境、动作或合法时长始终优先。未指定项由 `TOOLS/xdy_flow.py` 读取最近 12 次成功的 schema v2 运行：

1. 衣柜、正式环境和动作各自优先选择最久未使用项。
2. 并列项使用本次记录的随机 seed 决定。
3. 最近 12 次出现过的完全相同三项组合不得复用。
4. 动作默认时长为 `01→7 秒、02→6 秒、03→5 秒、04→6 秒`；用户可覆盖为 `5`、`6` 或 `7` 秒。

衣柜正文只取 `MATERIAL/anna-wardrobe.md` 对应条目的“款式提示词”；人物身份与基础身材比例不在穿搭段重复扩写。`其他` 段只补充写实摄影、稳定构图和物理一致性，其中服装颜色、面料、领型、袖型、层次、腰线位置与下装版型全程一致；人物身材体量与胸臀强化描述不在此段重复扩写。环境只从配置指定的正式 `anna-room-NN.png` 池中选择。

## TNS 版本

`v1` 使用首次锁定衣柜。只有 Dreamina 明确返回 TNS 且无 MP4 时才创建 `v2`–`v5`：每版仅更换衣柜并直接生成新的 `vid-prompt-vN.txt`；动作、环境、时长、双图顺序和随机 seed 继承本次内容锁定记录。每版均需单独通过 lint 和不可变生成合同。

## 校验

```bash
.venv/bin/python TOOLS/prompt_lint.py lint "TEMP/$RUN_ID/vid-prompt-v1.txt"
```

校验拒绝段落缺失、重复或乱序，非法双图引用，固定人物（含胸部体量一致性锚点）、环境或视频约束变化，prompt 内显式时长，内部流程词和其他不符合项目合同的内容。
