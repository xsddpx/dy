# 模块 01：参考、永久禁用检查与 grid prompt

## 职责

- 按当天 active 行程接收或选择参考，完成永久禁用检查和 6 帧参考宫格验证。
- 执行者查看宫格或帧图，反推可见导演结构。
- 写入 `TEMP/RUN_ID/grid-prompt.txt`；后续阶段只通过 `TOOLS/prompt_lint.py derive` 从它派生 prompt。

## 核心原则

- 当前日期位于 active 行程窗口内时必须复用；资产缺失、无效、未开始或过期时，先生成今天起连续 7 天的国内真实城市行程并保存，再继续选择参考。
- 当天内容必须匹配行程资产中的日期、城市、季节天气依据、白天/晚上方向、活动、旅行状态、穿搭方向和参考主题关键词；行程地点只是大方向，不是具体参考地点或 prompt 环境的硬锁。
- 生成或重建一周行程资产时，每天不得写具体钟点，`time_slot` 统一写 `白天；晚上`；每天必须同时包含白天和晚上两个可拍方向，分别写清活动、可见场景、穿搭方向和参考关键词。需要更细分时可使用 `daytime_option` 和 `night_option`，但二者也不得写具体钟点。
- 参考优先级固定为：用户指定参考 > 抖音收藏中匹配当天行程大方向的参考 > 已收藏视频作者主页中匹配当天行程大方向的其它公开视频兜底参考。up 主视频兜底只在收藏没有合适主题时使用，仍必须做永久禁用检查和参考宫格验证。
- 用户明确要求以后不要使用的参考必须写入 `MATERIAL/reference-blocklist.json`；命中永久禁用账本时直接跳过。
- up 主视频兜底候选必须能从用户收藏页的已收藏视频追溯到该作者主页，并确认候选是该作者主页下的公开视频；无法确认收藏作者链路、作者主页或候选视频归属时，不得作为兜底候选。不得从泛热门、推荐流或搜索结果中直接取兜底参考。
- 同等可用参考中，按 `DOCS/PROJECT.md` 当前阶段性内容增长方向排序；该排序只用于候选取舍，不改变参考选择优先级。
- 永久禁用检查和参考宫格通过是候选参考选出后的验证门槛，不是选题排序信号；候选未通过时退回本模块重新选择参考。
- 具体参考和成片环境可以不同于行程主地点，但必须能解释为当天行程附近、途中或延展活动，例如酒店、机场、游泳馆、网球馆、商场、酒吧、夜店、街区、廊下、咖啡馆或交通中转空间。
- 无论收藏或 up 主兜底参考，都只复刻可见导演结构：场景、穿搭版型、构图、动作路径、镜头、表情节奏、节拍点和封面停顿；人物统一替换为 anna，不复刻真人身份、账号标识、字幕水印、品牌商标或专有 IP。
- 参考宫格和帧图只用于人工视觉分析，不作为 Kie 或 Dreamina 的视觉输入。
- 行程匹配、类型判断、卖点校准和参考取舍只用于内部判断；`grid-prompt.txt` 只写可直接生成的可见内容，不写分析字段、文件或来源、流程、合规、平台解释及“根据/融合/吸收”等表达。
- 不新增脚本拼接 prompt；执行者必须根据宫格或帧图人工写成一份连贯、无矛盾、可直接提交的最终文本。

## grid-prompt 固定结构

`grid-prompt.txt` 必须使用十段式结构，且段落顺序固定：

- `人物：`
- `视频类型：`
- `穿搭：`
- `姿态镜头：`
- `环境：`
- `卖点与锁定：`
- `表情节奏：`
- `整体动画：`
- `背景音乐：`
- `其他：`

每段冒号后都必须写最终可执行内容，不得保留省略号、尖括号占位、条件分支或模板说明。`人物：` 段统一写成确定身份锚定句，只写身份、五官、脸型、发型、自然神态和稳定身材比例，不写姿态、穿搭、环境或画面锚点。

确认图 img prompt 使用：

```text
人物：@图1 是同一位成年女性的多视角、多表情角色参考图，不是多人合照；以 @图1 中左下大脸和正面脸为主要身份依据，侧面、背面和表情小图只用于辅助保持发型、脸型、身材比例和整体气质。以 @图1 中的同一位成年女性作为身份、五官、脸型、发型、自然神态和稳定身材比例依据，保持同一人物的一致性。画面中只出现这一位成年女性。
```

slow 视频阶段上传的是用户确认后的 Kie 原始确认图，不再上传 `anna.png` 多视角角色卡；因此 slow 的 `vid-prompt-v1.txt` 从 `grid-prompt.txt` 派生时，必须只从 `人物：` 段删除上述“@图1 是同一位成年女性的多视角、多表情角色参考图...”角色卡声明，保留其余人物身份一致性文本和其他九段内容。

## 视频类型规则

- `视频类型：` 段必须写主类型、次类型和一句短的成片方向，例如 `视频类型：穿搭展示；次类型：健身运动；近景穿搭展示结合轻运动互动节奏。`。
- 主类型只能选择一个最贴近的类型：`舞蹈律动`、`健身运动`、`穿搭展示`、`镜前自拍`、`走路回头`、`坐姿互动`、`双人互动`、`生活场景剧情`。
- 次类型可选且最多一个；需要时从同一集合中选择，不需要时写 `次类型：无`。不得新增其他类型名。
- 类型判断提示：
  - `舞蹈律动`：音乐节拍驱动，身体、手臂、肩颈或髋部有连续律动，动作重点是节奏而非展示某件衣服。
  - `健身运动`：动作来自训练、拉伸、瑜伽、运动装备展示或健身房/运动空间，身体活动和运动状态是主线。
  - `穿搭展示`：镜头主要展示服装版型、上身轮廓、腰线、整体搭配和封面姿态，动作多为站姿、转身、整理衣服或轻互动。
  - `镜前自拍`：镜子、手机、自拍视角或镜中人物关系是主要构图来源。
  - `走路回头`：人物行走、靠近/远离、回头、转身或跟拍路径是主要动作线。
  - `坐姿互动`：人物坐在沙发、椅子、床边、车内等位置，动作以坐姿调整、手势、眼神互动为主。
  - `双人互动`：参考里有两人或多人互动；dy 仍只执行 anna 单人路线，后续必须把互动关系转译为单人可执行动作，不启用双人路线。
  - `生活场景剧情`：动作有清晰日常事件或情节目标，例如进门、拿东西、整理房间、做饭、通勤准备。

## 段落写作规则

- `人物：` 使用统一确定提示词，只负责身份、五官、发型、脸型、神态和稳定身材比例；确认图必须保留 anna 多视角角色卡声明，视频阶段必须删除该声明；不得把 `anna.png` 的穿搭、姿态、环境或画面锚点当作依据，穿搭、动作和场景必须来自本次参考分析。
- `视频类型：` 写主类型、次类型和一句短的成片方向，不写内部判断依据。
- `穿搭：` 必须直接来自宫格或帧图的可见画面分析，先锁定服装结构，再写材质与贴合；必须写清衣服类别、颜色、领口方向、左右肩臂露出关系、袖长、上衣长度、腰线、裙裤形态、面料贴合、外搭层次和画面裁切范围。
- 特殊或不对称穿搭必须写清左右结构：哪一侧露肩、哪一侧长袖、斜向领口从哪侧延伸到哪侧、腰线位置、下装左右差异、开口或露腿位置、裤腿或裙摆是否连续、袖口或裤腿到画面何处被裁切。
- 宫格中能看到但容易被模型改写的部位必须显式描述，例如袖口到手腕、单侧高开线条、另一侧长裤腿、胯部与大腿上部裁切、肩带或领口是否连续；参考不清晰时只写可确认部分，不脑补品牌、配饰或不可见层次。
- `姿态镜头：` 必须写清站坐姿、侧身或前倾、重心、身体朝向、手部位置、裁切、镜头距离、角度、固定或手持关系、运镜关系和封面姿态。
- `环境：` 必须写清空间类型、背景物、家具或道具、材质、光线来源、空间纵深和人物与环境关系，并与当天行程的城市、季节天气、白天/晚上方向和旅行状态合理一致；最终只写可见空间与光线，不写“行程资产”“根据行程”等来源说明。
- `卖点与锁定：` 必须写入穿搭版型、领口与上身轮廓、面料张力、腰线、腰胯比例、姿态、镜头裁切、动作重点和封面停顿；禁止只写“性感”“好身材”“动作自然”等泛化审美词。
- `表情节奏：` 必须参考可见画面，拆成开场、中段、结尾的眼神、嘴角、眉眼和脸部状态；参考不清晰时写自然平静、专注或放松，不主动脑补成微笑。微笑可以记录，但不得把“最后微笑”当作默认收束；重写、重生或 TNS 收敛 prompt 时同样不得惯性补“最后微笑”，结尾优先使用抬眼看镜头、轻偏头、回到平静表情、短暂停顿或继续手部动作等自然收束。
- 参考中的脸部可见程度、遮挡关系和看镜头关系必须按参考处理；如果参考本身是手机挡脸、侧脸、低头、背影或不看镜头，不得为了“脸部完整可见”刻意改写，除非用户当次明确要求露出脸。
- `整体动画：` 围绕约 5-6 秒成片重新编排连续动作、手部路径、身体重心变化、镜头距离、角度、运镜、节奏点和结尾停顿，不按参考原时长逐秒照搬。
- `背景音乐：` 只写音乐风格、节奏和情绪，不写歌词、对白、口播、环境声、脚步声、衣料声、镜头声、喘息或音效。
- `其他：` 必须加入写实画质约束：`皮肤真实，不要刻意磨皮美化，写实摄影风格，真实人物质感，自然光影，真实皮肤纹理，细腻毛孔，轻微皮肤瑕疵，真实面部结构，真实眼神反光，真实头发丝细节，真实服装材质，符合物理规律的光照和阴影，自然景深，真实镜头质感，真实环境透视，真实空气感，真实色彩，不夸张，不塑料感，不磨皮，不网红滤镜，不过度锐化，不AI感。` 同时写清穿搭轮廓、腰线可读、构图稳定、单一连续画面和画面物理真实；必须明确不要生成拼图、分屏、多格、上下或左右双画面、多姿势拼贴；不写合规说明或平台解释。

## 卖点表达规则

- 每次进入提示词前必须做身材卖点校准：成熟曲线、清晰腰线、上身动态辨识度、领口与上身轮廓、上胸线可读、修身面料张力、腰胯比例必须能被镜头读到；弱卖点方向不得进入生成或发布。
- 核心卖点只用可见画面语言表达，不写直白词：
  - “胸部大”固定转译为：`饱满的立体廓形`、`高感知度的面料张力`、`上身丰盈的沙漏型线条`、`领口与上身轮廓清晰`。
  - “屁股大/臀部明显”固定转译为：`腰胯比例明显`、`臀胯轮廓自然凸显`、`高腰线与下装包裹出稳定曲线`、`古典雕塑般的 S 形动态`。
- `卖点与锁定：` 必须覆盖姿态、镜头、场景、穿搭、身材描述、动作与封面停顿六项最终取舍，但只写成可执行画面语言，不写内部条目标题。

## prompt 派生规则

- slow 确认图 img prompt 必须由 `derive --mode slow-img` 生成，只保留 `人物：`、`视频类型：`、`穿搭：`、`姿态镜头：`、`环境：`、`卖点与锁定：`、`表情节奏：`、`其他：` 八段。
- 派生后的 img prompt 仍必须是 Kie 可直接执行的静态画面描述；如果删除动画和音乐后出现断裂、矛盾、动态动作过强，或缺少单一连续画面、不生成拼图/分屏/多格的约束，必须回到 `grid-prompt.txt` 重写对应段落。
- slow 视频 prompt 必须由 `derive --mode slow-vid` 生成，只删除 `人物：` 段里关于 `@图1` 是多视角、多表情角色参考图的角色卡声明。
- 删除后，`人物：` 段仍必须保留同一位成年女性、身份、五官、脸型、发型、自然神态、稳定身材比例和单人画面约束。
- 除删除角色卡声明外，slow 视频阶段不得改写、补写、重组其他人物描述或其他九段内容。

## 命令

```bash
python3 TOOLS/reference_dedupe.py check "REFERENCE_URL"
python3 TOOLS/browser_reference_grid.py "REFERENCE_URL" --out-dir TEMP/RUN_ID/ --record-jsonl TEMP/RUN_ID/RUN_ID-run-record.jsonl
python3 TOOLS/prompt_lint.py derive TEMP/RUN_ID/grid-prompt.txt --mode slow-img --out TEMP/RUN_ID/A-01-img-prompt-v1.txt
python3 TOOLS/prompt_lint.py derive TEMP/RUN_ID/grid-prompt.txt --mode slow-vid --out TEMP/RUN_ID/vid-prompt-v1.txt
```

## 通过标准

- active 行程有效且当天大方向已用于参考主题匹配。
- 候选参考未命中 `MATERIAL/reference-blocklist.json`。
- `reference-grid.jpg`、两种宫格报告和十段式 `grid-prompt.txt` 已写入 `TEMP/RUN_ID/`。
- 宫格报告 `decision=pass` 且 `validation.decision=pass`。
- 确认图和视频阶段 prompt 均由 `prompt_lint.py derive` 生成并通过校验。
- 宫格只用于分析与记录，不作为生成接口的视觉输入。
