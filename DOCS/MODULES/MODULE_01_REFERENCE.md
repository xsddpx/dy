# 模块 01：参考、去重与宫格

## 职责

- 接收用户指定参考，或从抖音收藏抽样。
- 对自主参考执行 7 天去重。
- 生成 6 帧参考宫格和报告。
- 根据参考宫格或帧图反推可见导演结构，写入 `grid-prompt.txt`。

## grid-prompt 写作规范

- 找到收藏参考视频后，本次创意以复刻可见导演结构为主：先反推参考视频的场景、穿搭版型、构图、动作顺序、手部和身体路径、镜头距离、角度、运镜、表情节奏、节拍点和封面停顿；正式生成尽可能与参考视频一致。
- 默认只把人物身份替换为 `anna`；不复刻原视频人物身份、真人脸、账号标识、字幕水印、品牌商标或可能侵权的专有 IP。
- 复刻只针对平台安全的可见导演元素；正式 prompt 只写可见画面、动作、镜头、场景、穿搭和画面质感，不写合规说明。
- 每次写提示词或选择方向前先核对账号核心卖点：成人女性魅力、修身穿搭、成熟曲线、清晰腰线、上身辨识度、生活化动作、眼神互动和克制吸引力；弱卖点方向不得进入生成或发布。
- 每次进入提示词前必须做身材卖点校准：成熟曲线、清晰腰线、上身动态辨识度、领口和上胸线可读、修身面料张力必须能被镜头读到。内部分析可直接记录大胸、乳沟、诱惑等增长判断；写入 TapNow 或正式提示词时统一转译为具体可见的画面表达，如低领、方领、吊带、修身上衣、上胸线、肩颈线条、腰线、面料贴合和完整着装。抖音文案仍需保持平台可发布表达。
- `grid-prompt.txt` 必须完成参考六锁定，并把结论写入正式记录：
  - 姿态锁定：站姿、坐姿、半躺、侧身、前倾、重心、身体朝向、手臂和手部停留位置。
  - 镜头锁定：俯拍、平拍、仰拍、近景、超近景、半身、全身、裁切位置、固定机位或手持、跟拍。
  - 场景锁定：房间类型、背景物、家具、桌面、沙发、镜子、门窗、墙面、光线和人物与道具的空间关系。
  - 穿搭锁定：衣服类别、颜色、领口、肩臂露出、上衣长度、腰线位置、裙裤形态、面料贴合和外搭层次。
  - 身材描述锁定：参考里实际可见的上身视觉体量、肩颈线、上胸线、腰线、腰胯比例、腿部入镜范围、曲线来源和被衣服、姿态、镜头放大的部位。
  - 动作与封面锁定：开场姿态、动作路径、手部路径、节奏、最后停顿和最适合做封面的瞬间。

## 命令

```bash
python3 TOOLS/reference_dedupe.py check "REFERENCE_URL" --window-days 7
python3 TOOLS/browser_reference_grid.py "REFERENCE_URL" --out-dir TEMP/RUN_ID/ --record-jsonl TEMP/RUN_ID/RUN_ID-run-record.jsonl
```

## 通过标准

- `reference-grid.jpg`、`reference-grid-report.json`、`reference-grid-report.md` 已写入 `TEMP/RUN_ID/`。
- `grid-prompt.txt` 已写入 `TEMP/RUN_ID/`，内容包括可见导演结构、人物动作连续性、镜头运动、节奏、姿态变化、画面动态重点、身材卖点校准和参考六锁定结论。
- 宫格报告 `decision=pass` 且 `validation.decision=pass`。
- 宫格用于 img prompt 和 grid prompt 分析与记录，也作为 Dreamina 视频生成的参考视觉输入；不得直接作为 Kie 确认图输入。
