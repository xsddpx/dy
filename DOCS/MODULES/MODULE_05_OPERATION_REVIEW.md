# 模块 05：运营复盘与选题决策

## 文档边界

- 本文是运营复盘、数据读取、指标归类和选题决策的唯一执行流程文档。
- `DOCS/PROJECT.md` 只保存阶段性内容增长结论；不存放复盘执行步骤、字段清单或页面操作流程。
- 默认日更发布任务不读取、不执行本文；只有用户明确要求复盘、查看作品表现、分析播放涨粉或更新内容方向时，才读取并执行本文。
- 本文只用于校准后续选题，不是 `auto/fast` 发布流程的硬节点。

## 职责

- 读取抖音创作者中心可见的账号、作品和数据中心指标。
- 统计近期作品的播放、点赞、评论、分享、收藏、完播、2s 跳出、平均播放时长和粉丝数变化。
- 按题材、场景、穿搭、动作钩子和发布时间归类高表现与低表现样本。
- 输出下一阶段内容方向和下一条视频选题建议。
- 将阶段性策略反馈到 `DOCS/PROJECT.md`，但不把单次复盘结论写成永久规则。

## 触发条件

- 用户明确要求查看作品表现、播放量、热度、涨粉或内容方向。
- 累计新增 10 条已发布作品后需要刷新阶段性策略。
- 连续 3 条作品明显低于近期播放中位数，且需要调整参考选择或题材轮换。
- 计划改变主线题材、场景池、标题标签策略或发布频率前。

## 数据来源

- 创作者中心作品管理页：发布时间、可见状态、标题、时长、播放、点赞、评论、分享、收藏。
- 创作者中心数据中心：播放中位数、条均点赞、条均评论、条均分享、平均播放时长、2s 跳出率、5s 完播率、点击率、投稿量。
- 账号信息：粉丝数、总获赞数和账号昵称。
- 本地运行记录：`TEMP/RUN_ID/RUN_ID-run-record.*`、`OUTPUT/RUN_ID.mp4`、发布报告和参考来源。

只使用已经可见或可导出的真实数据；页面未显示的指标不得猜测。无法读取粉丝增量时，只报告当前粉丝数，并说明缺少增量来源。

## 样本范围

- 复盘默认尽量覆盖创作者中心当前可见的历史作品范围，不只看最近几条或播放最高的前几条。
- 页面一次只能展示部分作品时，应继续翻页或滚动收集，直到达到页面可见上限、用户指定范围，或数据已经足够覆盖主要发布阶段。
- 输出必须说明实际样本范围，例如 `近 34 条投稿`、`当前页面可见 50 条` 或 `2026-06-01 至 2026-06-26 可见作品`。
- 样本分析必须分层：高表现、中位区间、低表现和近期新发都要看；不得只用头部爆款推导下一阶段方向。
- 历史样本较多时，优先按时间阶段、题材类型、场景类型、穿搭关键词和动作钩子分组，再比较播放、留存和互动。
- 新近发布但数据尚未稳定的作品可以单独列为观察样本，不得直接并入成熟表现结论。

## 记录字段

发布记录或本地运行上下文可得时，可以读取以下基础字段用于复盘归类：

- `published_url`
- `publish_time`
- `title`
- `description`
- `tags`
- `reference_url`
- `reference_video_id`
- `content_type`
- `scene_type`
- `outfit_keyword`
- `hook_action`
- `duration_sec`
- `ai_generated_declaration`

复盘阶段读取到作品表现时，再追加或单独记录以下指标：

- `play_count`
- `like_count`
- `comment_count`
- `share_count`
- `favorite_count`
- `average_play_duration`
- `completion_rate_5s`
- `bounce_rate_2s`
- `cover_click_ratio`
- `follower_count`
- `follower_delta`

指标缺失时保留为空或不记录，不得用估算值补齐。

## 归类维度

- `content_type`：场景化穿搭、镜前穿搭、运动卡点、日常 fitcheck、旅行外景、室内氛围。
- `scene_type`：酒店窗边、小酒馆、海边、泳池、公园、电梯口、门边、客厅、白墙、镜前、交通中转等。
- `outfit_keyword`：瑜伽裤、低腰、牛仔短裤、短裙、修身针织、黑色修身、粉裙、开衫等。
- `hook_action`：回头、靠近镜头、整理头发、整理衣服、抬头看镜头、坐姿调整、走近、转身、结尾停顿。
- `retention_signal`：平均播放时长、5s 完播率、2s 跳出率。
- `interaction_signal`：赞播比、评播比、分享率、收藏率。

## 判断规则

- 不只按播放量排序；播放量、留存和互动必须一起看。
- 不只看前几条高播放作品；必须结合中位样本、低表现样本和近期新发样本判断内容方向。
- 高播放但 2s 跳出过高的样本只说明封面或推荐入口强，不直接证明内容结构可复制。
- 低播放但 5s 完播率、平均播放时长或赞播比高的样本，可作为二次优化方向。
- 题材复用必须结合账号当前阶段；单条异常高播放不得直接覆盖主线策略。
- 私密、未发布、审核中或指标为 `-` 的作品只作为内容题材参考，不进入表现统计。
- 复盘结论必须标注数据日期和样本范围，例如 `2026-06-26，近 34 条投稿`。

## 更新边界

- 可以更新 `DOCS/PROJECT.md` 的阶段性内容增长方向。
- 可以建议调整模块 01 的同等参考排序信号。
- 不直接修改角色身份、默认模式、关键节点、TNS 收敛、发布声明或 prompt 十段式结构。
- 不因运营复盘结果自动发布、删除、私密或重新公开任何作品。
