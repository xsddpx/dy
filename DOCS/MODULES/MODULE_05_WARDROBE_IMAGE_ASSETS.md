# 模块 05：衣柜图片化入库

## 职责

本模块根据用户本次提供的一张服装参考图，生成一张穿在无头、中性、哑光展示人台上的标准电商商品图，同时编写与图片一一对应的服装描述，并将两者作为一个衣柜条目入库。

本模块不生成人物穿搭图或视频，不触发 xdy、xdysp、Dreamina、Google Drive 或平台发布。最终回复只交付已入库的人台商品图。

## 输入与生成合同

- 必须由用户本次提供参考图，不从历史会话、旧运行目录或现有衣柜自行选图。
- 参考图是服装事实的唯一来源。真人身份、脸部、头发、妆容、身材、姿势、手机、界面、文字、场景和道具均不属于服装事实。
- 目标是参考图中可见的整套服装；上装、下装、裙装、领带、腰带和袜类等明确穿搭组件必须完整保留。
- 原图不足以确认核心结构或目标组件时停止，请用户补图或说明，不自行推断。
- 固定使用 `$imagegen` 技能的内置图片生成工具；每轮只使用原始参考图，失败候选不得成为下一轮输入。

固定基础 prompt：

```text
根据参考图中的服装，生成一张标准电商服装商品图。将整套服装穿在无头、中性、哑光的服装展示人台上；人台正面自然站立，只用于撑出版型，不具有脸部、头发、妆容或真人身份特征。准确保留参考图中可见的服装组件、颜色、版型、领口或肩带、层次、开合、腰线、裙裤轮廓、长度、图案、面料观感、褶皱和袜类结构。固定使用 9:16 竖版正面全身构图，人台居中、完整，服装主体占据画面主要区域，四周保留均匀少量留白；使用纯白色或浅灰色干净背景和均匀柔和棚拍光线。不要添加文字、品牌、参数、装饰、手机、界面、场景道具或参考图中没有的设计。
```

## 图片硬门

入库前逐项对照原始参考图：

1. 参考图中的目标服装组件全部存在。
2. 品类、颜色、领口或肩带、袖型、层次、开合、腰线、裙裤轮廓和长度一致。
3. 图案、面料观感、褶皱、缝线、蕾丝、透明度、光泽和袜类结构对应可见证据。
4. 画面为 9:16 正面全身商品图，服装完整清晰，人台没有遮挡、吞并或改变结构。
5. 只有一个无头中性哑光人台；没有真人特征、文字、品牌、手机、界面、道具或新增设计。

任一项不通过时丢弃结果，重新以原始参考图生成；只允许在基础 prompt 后追加一句针对当前错误的中性服装事实。

## 服装描述

图片通过硬门后，根据原始参考图和最终商品图共同编写一条 `款式提示词`：

- 使用单段中文，完整描述组件、颜色、面料、领型或肩带、袖型、层次、开合、腰线、下装轮廓、长度、图案和袜类结构。
- 只记录可见服装事实，不描述人物身份、身材、姿势、人台、背景、鞋履或生成过程。
- 使用明确、单一、无条件分支的表述，并以服装组件和结构全程一致、衣料仅自然形变的稳定性约束收尾。
- 必须通过模块 01 的 TNS 词汇和三图 prompt 校验；图片与描述不一致时不得入库。

## 原子入库

正式目录结构固定为：

```text
MATERIAL/wardrobe-images/
└── 衣柜图-001/
    ├── 衣柜图-001.png
    └── 服装描述.md
```

描述文件固定格式：

```markdown
# 衣柜图-001

- 图片：衣柜图-001.png
- 款式提示词：<完整单段服装描述>
```

入库规则：

- 只识别 `衣柜图-NNN/`，其中 `NNN` 为三位十进制序号。
- 空衣柜从 `001` 开始；否则取最大编号加一，不回填缺号、不复用删除编号。
- 序号只在商品图和描述全部通过后分配；失败候选不占号。
- 先在正式目录外准备并校验两个文件，再以目录重命名一次性入库；不得留下半条目。
- 目标目录存在时停止并重新扫描，禁止覆盖或原地修改已入库条目。
- 每个条目只能包含同编号 PNG 和 `服装描述.md`；不得保存参考图、候选图、代理图、prompt、JSON、状态文件或其他内容。
- 编号达到 `999` 时停止，未经用户修改合同不得改为四位编号。
- 入库后立即成为正式三图视频可选条目，不维护实验或验证状态。

## 最终交付

入库成功后，最终回复只使用本地绝对路径 Markdown 图片交付正式商品图：

```markdown
![衣柜图-001](/绝对路径/MATERIAL/wardrobe-images/衣柜图-001/衣柜图-001.png)
```

不附加原始参考图、候选图、代理图、prompt、模型状态、质检说明或路径列表。

## 与正式主链的关系

模块 01 通过 `TOOLS/select_wardrobe.py` 读取本模块的完整条目。图片固定作为 `@图2`，`服装描述.md` 的款式提示词原文紧接服装角色锚点写入 vid prompt；两者编号或事实不一致时硬阻断。
